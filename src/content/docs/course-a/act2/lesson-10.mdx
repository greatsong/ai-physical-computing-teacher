## 도입 (5분)

<div class="teacher-script">
  <p>"지금까지 우리가 해온 과정을 떠올려봅시다. 1막에서 직접 코드를 짜서 센서를 읽었고, 2막에서 바이브 코딩으로 복잡한 시스템을 만들었고, 지난 시간에는 데이터를 정리하고 그래프로 만들었어요."</p>
  <p>"그런데 그래프를 보고 '이 데이터가 무엇을 의미하는지' 해석하는 건 여전히 사람의 몫이었죠?"</p>
</div>

<div class="student-voice">
  <span class="bubble">"네, 그래프 보고 우리가 직접 판단했어요"</span>
  <span class="bubble">"AI한테 시키면 안 되나요?"</span>
  <span class="bubble">"AI가 패턴 찾는 건 잘하지 않아요?"</span>
</div>

<div class="teacher-script">
  <p>"좋은 질문이에요. 오늘은 정말로 AI에게 데이터를 보여주고 분석을 시켜봅니다."</p>
  <p>"그런데 중요한 건, AI의 분석을 <strong>그대로 믿는 게 아니라</strong>, 맞는지 틀린지 <strong>검증</strong>하는 거예요. AI가 뭘 잘하고 뭘 못하는지, 오늘 직접 확인합니다."</p>
  <p>"참고로 이번 시간은 2막의 마지막 차시입니다. 바이브 코딩에서 데이터 분석까지, 2막 전체의 흐름을 정리하는 시간이기도 해요."</p>
</div>

<div class="teaching-tip">
  <p>"AI에게 시키면 되지 않느냐"는 질문이 나오면 좋은 출발점입니다. 이 수업의 핵심은 AI를 <strong>맹목적으로 신뢰하지 않는 태도</strong>를 기르는 것입니다. AI의 분석을 받은 뒤 "이게 정말 맞아?"라고 묻는 습관이 이 차시의 목표입니다. 2막 마지막 차시이므로 5&#126;8차시의 바이브 코딩 경험과 연결해 주세요.</p>
</div>

## 전개 (35분)

<div class="step-header">
  <span class="step-num">1</span>
  <span class="step-title">데이터를 AI에게 전달할 형태로 준비</span>
  <span class="step-time">7분</span>
</div>

<div class="teacher-script">
  <p>"먼저 학생 교재의 기본 코드를 PC Python에서 실행하세요. 이 코드는 센서 데이터를 AI에게 보여줄 수 있는 텍스트 형태로 정리해줍니다."</p>
  <p>"실행하면 콘솔에 표 형태로 데이터가 출력됩니다. 측정 조건, 데이터, 기본 통계, 환기 전후 비교까지 포함되어 있어요. 이 전체를 복사하세요."</p>
  <p>"자기 데이터가 있는 사람은 코드의 샘플 데이터 대신 자기 데이터를 넣어도 좋습니다."</p>
</div>

<div class="student-voice">
  <span class="bubble">"표가 깔끔하게 나왔어요"</span>
  <span class="bubble">"환기 전후 평균이 이미 계산되어 있네요"</span>
  <span class="bubble">"전체 선택해서 복사했어요"</span>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"코드 실행이 안 돼요"</div>
    <span class="label">원인</span>
    <span class="value">인터프리터가 MicroPython으로 설정되어 있음</span>
    <span class="label">대응</span>
    <span class="value">지난 시간처럼 "Local Python 3"으로 전환. 이 코드는 외부 라이브러리가 필요 없어 순수 Python으로 실행됩니다</span>
  </div>
</div>

<div class="teaching-tip">
  <p>이 단계의 코드는 pandas 없이 순수 Python으로 동작합니다. 9차시에서 라이브러리 설치에 어려움이 있었던 학생도 문제없이 실행할 수 있습니다.</p>
</div>

<div class="step-header">
  <span class="step-num">2</span>
  <span class="step-title">AI에게 분석 요청하기</span>
  <span class="step-time">10분</span>
</div>

<div class="teacher-script">
  <p>"자, AI 챗봇을 열어주세요. 복사한 데이터를 붙여넣고, 분석을 요청합니다."</p>
  <p>"그런데 여기서 중요한 포인트가 하나 있어요. 같은 데이터라도 <strong>어떻게 물어보느냐</strong>에 따라 AI의 답이 완전히 달라집니다."</p>
  <p>"학생 교재에 프롬프트 예시가 4가지 있어요. 패턴 찾기, 환기 효과 분석, 비교 분석, 개선 제안. 최소 2가지 다른 프롬프트로 물어보고, AI 답변이 어떻게 달라지는지 비교하세요."</p>
  <p>"특히 <strong>맥락 정보</strong>를 얼마나 포함하느냐에 주목하세요. '이 데이터 분석해줘'만 보내는 것과, '25명이 있는 교실에서 5분째에 환기를 시작했다'고 보내는 것 — AI 답변이 어떻게 다른지 직접 실험해보세요."</p>
</div>

<div class="student-voice">
  <span class="bubble">"AI가 환기 효과를 수치로 설명해줬어요!"</span>
  <span class="bubble">"맥락 없이 보내니까 답변이 너무 일반적이에요"</span>
  <span class="bubble">"맥락을 넣으니까 훨씬 구체적으로 분석해줘요"</span>
  <span class="bubble">"같은 데이터인데 프롬프트 바꾸니까 다른 관점으로 분석해요"</span>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"AI 챗봇에 접속이 안 돼요"</div>
    <span class="label">원인</span>
    <span class="value">학교 네트워크 차단 또는 로그인 문제</span>
    <span class="label">대응</span>
    <span class="value">교사 계정으로 시연하거나, 학생 2명이 한 컴퓨터를 공유하게 하세요. 사전에 네트워크 허용 여부를 확인하는 것이 중요합니다</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"AI 답변이 너무 짧아요"</div>
    <span class="label">원인</span>
    <span class="value">프롬프트가 너무 간단함 ("분석해줘"만 보낸 경우)</span>
    <span class="label">대응</span>
    <span class="value">맥락 정보와 구체적인 분석 요청을 추가하도록 안내. "패턴을 찾고, 이상값을 지적하고, 개선 제안을 해줘"처럼 구체적으로</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>순회 시 확인:</strong></p>
  <ol>
    <li>최소 2가지 다른 프롬프트로 질문하고 있는가</li>
    <li>맥락 정보 포함 여부에 따른 답변 차이를 인식하고 있는가</li>
    <li>AI 답변을 그대로 수용하지 않고 의문을 제기하는가</li>
  </ol>
</div>

<div class="teaching-tip">
  <p>프롬프트의 품질이 AI 분석의 품질을 결정합니다. 국어 시간의 <strong>논증 구조</strong>와 연결할 수 있습니다 — "주장에 근거를 붙이듯, AI에게도 맥락과 조건을 붙여야 좋은 답을 받는다." 이 연결은 S등급 학생에게 특히 효과적입니다.</p>
</div>

<div class="step-header">
  <span class="step-num">3</span>
  <span class="step-title">AI 분석 결과 검증하기</span>
  <span class="step-time">10분</span>
</div>

<div class="teacher-script">
  <p>"AI 답변을 받았으면, 이제 <strong>검증</strong>할 시간입니다. AI가 맞게 분석한 것과 틀리게 분석한 것을 구분하세요."</p>
  <p>"예를 들어, AI가 '환기 후 CO2가 30% 감소했다'고 했다면, 그 숫자가 맞는지 여러분이 직접 확인해보세요. 9차시에서 계산한 것과 비교해도 좋고요."</p>
  <p>"특히 주목할 점은 — AI가 <strong>교실 크기, 학생 수, 계절, 창문 위치</strong> 같은 걸 모른다는 거예요. 그런데 AI가 마치 다 아는 것처럼 분석하진 않았나요? 이걸 찾아보세요."</p>
</div>

<div class="student-voice">
  <span class="bubble">"AI가 교실 크기를 모르면서 환기량을 추정했어요"</span>
  <span class="bubble">"계절도 모르면서 냉방 제안을 했어요"</span>
  <span class="bubble">"수치 계산은 정확했는데, 원인 추측은 틀렸어요"</span>
  <span class="bubble">"확신에 차서 말하니까 맞는 줄 알았어요"</span>
</div>

<div class="teaching-tip">
  <p><strong>"확신에 찬 오류"</strong>가 이 차시의 핵심 발견입니다. AI가 틀린 분석을 자신 있는 말투로 하는 사례를 학생이 직접 찾아내면, 비판적 사고가 체화됩니다. "AI가 자신 있게 말하면 맞는 건가요?"라고 질문하세요.</p>
</div>

<div class="step-header">
  <span class="step-num">4</span>
  <span class="step-title">AI vs 나 비교 토론</span>
  <span class="step-time">8분</span>
</div>

<div class="teacher-script">
  <p>"모둠에서 토론해봅시다. 세 가지 질문이에요."</p>
  <p>"첫째, AI가 나보다 더 잘한 것은 뭐예요? 아마 수치 계산이나 패턴 인식 같은 거겠죠."</p>
  <p>"둘째, 내가 AI보다 더 잘 판단한 것은? '우리 교실은 3층이고 창문이 북향이라 겨울에 춥다' — 이런 맥락은 내가 아는 거잖아요."</p>
  <p>"셋째, 데이터 수집 → 정리 → AI 분석, 이 전체 과정에서 각 단계의 주인공은 누구인가요? 센서? 코드? AI? 사람?"</p>
</div>

<div class="student-voice">
  <span class="bubble">"AI는 계산이 빠르고, 나는 상황을 아는 게 장점이에요"</span>
  <span class="bubble">"결국 사람이 판단해야 해요"</span>
  <span class="bubble">"AI에게 좋은 질문을 하는 게 중요한 것 같아요"</span>
  <span class="bubble">"사람 + 코드 + AI가 같이 해야 제대로 되네요"</span>
</div>

<div class="patrol-point">
  <p><strong>순회 시 확인:</strong></p>
  <ol>
    <li>AI 활용 평가표를 작성하고 있는가</li>
    <li>"AI 장점 / AI 한계 / 사람의 역할"을 구분하고 있는가</li>
    <li>2막 전체 흐름(바이브 코딩 → 데이터 수집 → 전처리 → AI 분석)을 인식하는가</li>
  </ol>
</div>

<div class="teaching-tip">
  <p>"사람 + 코드 + AI가 각자의 강점을 발휘할 때"라는 결론이 자연스럽게 나오도록 유도하세요. 이것이 2막 전체의 핵심 교훈입니다. 시간이 남으면 학생 교재의 인사이트 대시보드 종이 활동을 진행하세요.</p>
</div>

## 정리 (10분)

<div class="teacher-script">
  <p>"오늘 수업을 정리해볼까요? 이 그림을 한번 보세요."</p>
  <p>"<strong>센서</strong>는 이상값을 보냅니다. <strong>코드</strong>는 맥락을 모릅니다. <strong>AI</strong>는 교실의 실제 상황을 모릅니다. <strong>사람</strong>은 수천 개의 숫자를 빠르게 분석할 수 없습니다."</p>
  <p>"그런데 이 네 가지가 합쳐지면? 센서가 수집하고, 코드가 정리하고, AI가 패턴을 찾고, 사람이 최종 판단합니다. <strong>각자 혼자서는 불완전하지만, 함께하면 의미 있는 결과</strong>가 나옵니다."</p>
  <p>"그리고 이것이 바로 2막 전체의 이야기입니다."</p>
</div>

<div class="teacher-script">
  <p>"2막을 돌아봅시다. 5차시에서 바이브 코딩을 배웠고, 6&#126;8차시에서 센서 시스템을 만들고 데이터를 모았고, 9차시에서 데이터를 정리하고 그래프를 그렸고, 오늘 10차시에서 AI에게 분석을 시키고 검증했습니다."</p>
  <p>"이 흐름에서 가장 중요한 것은 뭘까요? <strong>AI를 쓸 줄 아는 것</strong>이 아니라, <strong>AI가 맞는지 판단할 줄 아는 것</strong>입니다. 그러려면 기본 지식이 있어야 해요. 센서가 어떻게 동작하는지, 데이터가 뭘 의미하는지 알아야 AI의 실수도 잡을 수 있죠."</p>
  <p>"다음 시간부터는 <strong>3막 '프로젝트'</strong>가 시작됩니다. 1막에서 배운 직접 코딩, 2막에서 배운 바이브 코딩과 데이터 분석을 모두 활용해서, 팀 프로젝트로 종합 대시보드에 도전합니다. 첫 번째 단계는 '<strong>측정 가능한 좋은 질문</strong>'을 세우는 것이에요."</p>
</div>

<div class="teaching-tip">
  <p>2막 마무리이므로 <strong>전체 흐름 정리</strong>에 충분한 시간을 할애하세요. 칠판에 "5차시 → 6차시 → ... → 10차시"를 쓰고, 각 차시의 핵심 키워드를 학생들에게 물어보며 채워나가면 효과적입니다. 3막 예고에서 "측정 가능한 좋은 질문"이라는 표현은 호기심을 자극합니다.</p>
</div>

## 차별화 전략

<div class="differentiation">
  <div class="fast-learner">
    <p><strong>빠른 학생에게:</strong></p>
    <ul>
      <li>동일한 데이터에 3가지 다른 프롬프트(패턴 탐색 / 인과 분석 / 예측)를 작성하여 AI 응답을 비교하기</li>
      <li>AI의 잘못된 추론을 발견하고, 보완 정보를 추가한 프롬프트로 개선 검증하기</li>
      <li>국어 시간의 논증 구조와 연결 — "근거 없는 주장"을 AI 분석에서 찾기</li>
      <li>반 전체에 자신의 프롬프트 전략을 공유하기</li>
    </ul>
  </div>
  <div class="slow-learner">
    <p><strong>느린 학생에게:</strong></p>
    <ul>
      <li>교사가 제시한 프롬프트 예시를 그대로 사용하여 AI에게 분석 요청</li>
      <li>AI 답변에서 "장점 1가지, 한계 1가지"만 찾기</li>
      <li>AI가 교실 상황을 모른다는 것을 하나의 사례로 확인하기</li>
      <li>짝과 함께 AI 활용 평가표 작성하기</li>
    </ul>
  </div>
</div>
