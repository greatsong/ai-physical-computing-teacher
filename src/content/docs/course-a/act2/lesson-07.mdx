<div class="prep-checklist">
  <h5>하드웨어 (팀별 1세트)</h5>
  <ul>
    <li>Raspberry Pi Pico 2 WH + Grove Shield</li>
    <li>심박 센서 (Pulse Sensor, 아날로그) — A0 포트(GP26)에 연결</li>
    <li>OLED 디스플레이 SSD1306 (I2C) — I2C 포트에 연결</li>
    <li>Grove 케이블 2개</li>
    <li>USB 데이터 케이블</li>
  </ul>
  <h5>소프트웨어</h5>
  <ul>
    <li>Thonny IDE + MicroPython 펌웨어</li>
    <li>ssd1306 라이브러리 설치 확인</li>
    <li>AI 챗봇 접속 확인</li>
  </ul>
  <h5>수업 전 확인</h5>
  <ul>
    <li>심박 센서를 교사가 미리 테스트 — 손가락 접촉 압력에 따라 신호 차이가 큼</li>
    <li>THRESHOLD 값의 적정 범위를 파악 (보통 40000&#126;50000 사이)</li>
    <li>교실에 제자리 뛰기를 할 수 있는 공간이 있는지 확인</li>
    <li>심박수 기록표, 모둠원 비교표 인쇄</li>
    <li>손이 차가운 학생이 많은 계절이면 손 비비기 안내 준비</li>
  </ul>

## 전개 (35분)

<div class="step-header">
  <span class="step-num">1</span>
  <span class="step-title">센서 연결 + AI에게 코드 요청</span>
  <span class="step-time">12분</span>
</div>

<div class="teacher-script">
  <p>"심박 센서를 <strong>A0 포트</strong>에, OLED를 <strong>I2C 포트</strong>에 연결하세요. 오늘은 장치가 2개라서 연결이 간단합니다."</p>
  <p>"연결됐으면 AI 챗봇을 열고 프롬프트를 보내세요. 교재의 프롬프트 예시를 참고하되, 핵심 정보를 반드시 포함하세요 — ADC 핀 번호, 샘플링 간격, THRESHOLD, OLED 주소."</p>
  <p>"AI가 코드를 주면, 세 가지를 먼저 확인하세요. 첫째, <code>ADC(Pin(26))</code>인가요? A0 포트니까 GP26이어야 합니다. 둘째, 샘플링 간격이 20ms 정도인가요? 셋째, 피크 감지에 THRESHOLD가 있나요?"</p>
</div>

<details class="student-content">
  <summary>학생 교재 — 프롬프트 예시</summary>
  <div class="content">

```
Pico 2 WH에 Grove Pulse Sensor(심박 센서)를 A0 포트(GP26)에,
OLED SSD1306을 I2C0(SDA=GP8, SCL=GP9)에 연결했어.

다음 기능을 하는 MicroPython 코드를 만들어줘:

1. 심박 센서를 20ms 간격으로 빠르게 읽기 (ADC)
2. 피크 감지 알고리즘으로 심장 박동 횟수를 세기
3. 10초마다 BPM(분당 심박수)을 계산
4. OLED 대시보드에 표시:
   - 상단: 실시간 심박 파형 그래프 (최근 64개 샘플을 그래프로)
   - 하단: BPM 숫자 + 상태(정상 60&#126;100, 빠름 100+, 느림 60-)
5. Shell에도 BPM을 출력

피크 감지는 THRESHOLD 값을 넘을 때 카운트하고,
같은 피크를 두 번 세지 않도록 MIN_GAP(300ms) 설정해줘.
```

  </div>
</details>

<div class="patrol-point">
  <p><strong>이 시점에 순회하며 확인:</strong></p>
  <ol>
    <li>심박 센서가 A0 포트에 연결되어 있는가 (I2C 포트에 꽂으면 동작 안 함)</li>
    <li>손가락을 센서에 올바르게 올려놓고 있는가 (너무 세게 누르지 않는가)</li>
    <li>OLED에 파형이 움직이고 있는가</li>
    <li>BPM 값이 합리적인 범위(60&#126;120)인가</li>
  </ol>
</div>

<div class="teaching-tip">
  <p>심박 센서는 학생마다 THRESHOLD가 다릅니다. 이것을 <strong>캘리브레이션</strong>이라고 합니다. "왜 사람마다 THRESHOLD가 달라야 할까요?"라고 물으면, 피부 두께, 혈류량, 손가락 크기 등의 차이로 자연스럽게 연결됩니다. 의료기기가 환자에 맞게 보정하는 것과 같은 원리입니다.</p>
</div>

## 정리 (10분)

<div class="teacher-script">
  <p>"자, 모둠원끼리 데이터를 비교해 봤죠? 안정 시 BPM, 운동 직후 BPM, 회복 시간 — 사람마다 다르죠?"</p>
  <p>"'회복이 빠른 사람은 체력이 좋다'라는 말이 있는데, 오늘 데이터로 보면 어떤가요?"</p>
  <p>"오늘 수업에서 가장 중요한 포인트가 하나 더 있어요. <strong>THRESHOLD 조절</strong>이요. 같은 코드를 돌려도 사람마다 THRESHOLD를 다르게 설정해야 했죠? 이것을 <strong>캘리브레이션(보정)</strong>이라고 합니다."</p>
  <p>"병원에서 혈압계를 쓸 때, 체중계를 쓸 때, 모든 측정 장비는 보정을 합니다. 여러분이 오늘 한 THRESHOLD 조절도 같은 원리예요."</p>
  <p>"정리하면, 오늘 여러분은 바이브 코딩으로 심박 대시보드를 만들었고, 운동 전후 심박수를 데이터로 비교했고, 센서 보정의 개념을 경험했습니다."</p>
  <p>"다음 시간에는 <strong>초음파 센서 + 부저 + LED Bar</strong>를 조합한 스마트 알림 시스템을 만듭니다. 거리에 따라 소리와 빛이 달라지는 주차 보조 시스템 같은 걸 만들어 볼 거예요."</p>
</div>

<div class="teaching-tip">
  <p>"캘리브레이션"이라는 개념이 이 수업의 숨은 주제입니다. 단순히 코드를 돌리는 것이 아니라, <strong>측정 장비를 본인에게 맞게 보정하는 경험</strong>이 과학적 소양과 직결됩니다. 시간이 남으면 "스마트워치도 처음 착용할 때 보정 과정이 있는데, 원리가 같아요"로 마무리하면 효과적입니다.</p>
</div>

## 차별화 전략

<div class="differentiation">
  <div class="fast-learner">
    <p><strong>빠른 학생에게:</strong></p>
    <ul>
      <li>안정&#8211;운동직후&#8211;1분&#8211;2분 회복의 BPM 데이터를 수집하여 팀원 4명의 회복 곡선을 비교해 보세요</li>
      <li>THRESHOLD 캘리브레이션이 사람마다 다른 이유를 혈류량과 피부 두께로 설명해 보세요</li>
      <li>도전 코드(CSV 저장)를 실행하여 데이터를 파일로 남겨 보세요</li>
      <li>신호가 안 잡히는 친구에게 하드웨어+소프트웨어 양쪽 해결책을 안내해 보세요</li>
    </ul>
  </div>
  <div class="slow-learner">
    <p><strong>느린 학생에게:</strong></p>
    <ul>
      <li>학생 교재의 기본 코드 탭을 그대로 복사하여 실행하기</li>
      <li>THRESHOLD는 교사가 "45000으로 바꿔보세요"처럼 구체적 값을 안내</li>
      <li>운동 전후 BPM 비교 활동에 참여하고, OLED의 숫자 변화를 관찰하여 기록하기</li>
    </ul>
  </div>
