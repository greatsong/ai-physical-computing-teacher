<div class="step-header">
  <span class="step-num">2</span>
  <span class="step-title">이상값 확인과 제거</span>
  <span class="step-time">8분</span>
</div>

<div class="teacher-script">
  <p>"그래프를 보세요. 회색 점선이 원본 데이터예요. 위로 튀어나온 점이 보이죠? 그게 9999ppm이에요. 교실에서 CO2가 9999ppm이면 어떻게 될까요?"</p>
</div>

<div class="student-voice">
  <span class="bubble">"숨을 못 쉬겠죠?"</span>
  <span class="bubble">"그 정도면 교실이 아니라 화산 분화구예요"</span>
  <span class="bubble">"센서가 오류를 낸 거예요"</span>
</div>

<div class="teacher-script">
  <p>"맞아요, 센서 오류예요. 이런 값을 <strong>이상값(outlier)</strong>이라고 합니다. 코드에서 <code>df[(df['co2'] &gt;= 100) &amp; (df['co2'] &lt;= 5000)]</code> 이 부분이 '100 미만이거나 5000 초과인 값은 버려라'라는 뜻이에요."</p>
  <p>"이상값을 제거하니까 평균이 어떻게 바뀌었나요? 콘솔 출력을 비교해보세요."</p>
</div>

<details class="student-content">
  <summary>학생 교재 — 전처리 효과 비교표</summary>
  <div class="content">

| 단계 | 데이터 개수 | 평균 | 최솟값 | 최댓값 | 범위 |
|------|-----------|------|--------|--------|------|
| 원본 데이터 | | | | | |
| 이상값 제거 후 | | | | | |
| 이동평균(3) 적용 후 | | | | | |
| 이동평균(5) 적용 후 | | | | | |

  </div>
</details>

<div class="teaching-tip">
  <p>이상값 제거 전후의 평균 변화를 <strong>숫자로</strong> 확인하게 하세요. "쓰레기를 넣으면 쓰레기가 나온다(Garbage In, Garbage Out)"를 체감하는 순간입니다. 수학 시간의 '대푯값'과 연결하면 교과 통합 효과가 있습니다.</p>
</div>

<div class="step-header">
  <span class="step-num">3</span>
  <span class="step-title">이동평균 필터 실험</span>
  <span class="step-time">12분</span>
</div>

<div class="teacher-script">
  <p>"이상값을 제거했지만, 아직도 그래프가 들쭉날쭉하죠? 이건 센서의 작은 오차 때문이에요. 이걸 매끄럽게 만드는 방법이 <strong>이동평균(Moving Average)</strong>입니다."</p>
  <p>"<code>.rolling(window=3).mean()</code> — 이건 '3개씩 묶어서 평균을 구해라'라는 뜻이에요. 윈도우 크기를 3, 5, 10으로 바꿔가며 그래프가 어떻게 달라지는지 비교해보세요."</p>
  <p>"코로나 때 뉴스에서 '7일 이동평균'이라는 말을 들어본 사람? 그거랑 같은 원리입니다."</p>
</div>

<div class="student-voice">
  <span class="bubble">"3이면 약간 부드러워지고, 10이면 완전 매끈해져요"</span>
  <span class="bubble">"근데 10이면 환기 시작 시점이 늦게 보여요"</span>
  <span class="bubble">"부드러워지는 대신 반응이 느려지네요"</span>
  <span class="bubble">"코로나 7일 이동평균이 이거였어요?"</span>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"이동평균 그래프가 처음 몇 개가 빠져 있어요"</div>
    <span class="label">원인</span>
    <span class="value">rolling()의 특성 — window=5이면 앞 4개는 NaN (데이터 부족)</span>
    <span class="label">대응</span>
    <span class="value">정상 동작입니다. "왜 처음 몇 개가 없을까?" 질문으로 이동평균 원리를 이해하게 유도하세요</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>순회 시 확인:</strong></p>
  <ol>
    <li>윈도우 크기를 최소 2가지 이상 비교하고 있는가</li>
    <li>"부드러움 vs 반응 속도" 트레이드오프를 인식하고 있는가</li>
    <li>이동평균 비교표를 작성하고 있는가</li>
  </ol>
</div>

<div class="teaching-tip">
  <p>이동평균의 <strong>트레이드오프</strong>를 직관적으로 이해하게 하는 것이 핵심입니다. "부드러워지면 좋은 것 아닌가요?"라는 질문에 "환기를 시작한 정확한 시점을 알고 싶다면?"이라고 반문하세요. 윈도우가 클수록 부드럽지만, 급격한 변화를 놓치게 됩니다.</p>
</div>

## 정리 (10분)

<div class="teacher-script">
  <p>"오늘 여러분은 숫자 20개를 가지고 세 가지 작업을 했어요."</p>
  <p>"첫째, <strong>이상값 제거</strong> — 센서 오류를 걸러냈습니다. 둘째, <strong>이동평균</strong> — 노이즈를 줄여서 추세를 드러냈습니다. 셋째, <strong>시각화</strong> — 숫자를 그래프로 바꿔서 누구나 이해할 수 있게 만들었습니다."</p>
  <p>"이 세 단계를 합쳐서 <strong>데이터 전처리</strong>라고 합니다. 현실 세계에서 데이터 과학자가 업무 시간의 80%를 전처리에 쓴다는 말이 있어요. 그만큼 중요합니다."</p>
  <p>"pandas를 한마디로 설명하면 '<strong>엑셀인데 마우스 대신 코드로 조작하는 것</strong>'이에요. 코드로 하면 뭐가 좋을까요? 반복할 수 있고, 기록이 남고, 자동화할 수 있죠."</p>
  <p>"다음 시간에는 이렇게 정리한 데이터를 <strong>AI에게 보여주고 분석을 시킵니다</strong>. AI가 사람이 못 찾은 패턴도 찾아줄까요? 직접 확인해봅시다."</p>
</div>

<div class="teaching-tip">
  <p>"데이터 과학자의 80%가 전처리"라는 통계는 학생들에게 전처리의 중요성을 인식시키는 데 효과적입니다. pandas = "코드로 쓰는 엑셀"이라는 비유도 잘 와닿습니다. 다음 시간 예고에서 "AI에게 분석을 시킨다"는 말에 학생들이 흥미를 보일 것입니다.</p>
</div>

## 차별화 전략

<div class="differentiation">
  <div class="fast-learner">
    <p><strong>빠른 학생에게:</strong></p>
    <ul>
      <li>이상치 제거 기준을 "평균에서 표준편차의 2배 이상"으로 통계적으로 정의해보기 (수학 시간 산포도와 연결)</li>
      <li>이동평균 창 크기별 그래프를 겹쳐 그려 트레이드오프를 시각적으로 비교</li>
      <li>환기 시점에 주석(annotation)을 추가하여 출판 수준 그래프 만들기</li>
      <li>pandas를 "엑셀인데 마우스 대신 코드로"라고 비유하며 친구에게 설명해보기</li>
    </ul>
  </div>
  <div class="slow-learner">

      <li>이상값이 그래프를 어떻게 왜곡하는지 눈으로 확인하는 것에 집중</li>
      <li>윈도우 크기 하나만 바꿔보는 작은 실험</li>
