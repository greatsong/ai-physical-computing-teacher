## 전개 (35분)

<div class="step-header">
  <span class="step-num">1</span>
  <span class="step-title">센서 3개 연결 + I2C 스캔</span>
  <span class="step-time">7분</span>
</div>

<div class="teacher-script">
  <p>"오늘 가장 중요한 포인트는 <strong>연결</strong>입니다. 세 장치 모두 I2C 통신을 사용하는데, Grove Shield에 I2C 포트는 2개뿐이에요."</p>
  <p>"SCD41을 I2C0, DHT20을 I2C1에 꽂으세요. OLED는 — 여분 케이블이 있으면 Y-스플리터로 같은 포트에 연결하거나, UART1 포트에 연결해도 됩니다. 같은 I2C 버스이니까요."</p>
  <p>"연결이 끝나면, 먼저 I2C 스캔 코드를 실행해서 세 장치가 다 보이는지 확인합시다."</p>
</div>

<details class="student-content">
  <summary>I2C 스캔 코드</summary>
  <div class="content">

```python
from machine import I2C, Pin
i2c = I2C(0, sda=Pin(8), scl=Pin(9))
devices = i2c.scan()
print("발견된 I2C 장치:", [hex(d) for d in devices])
# 예상: ['0x38', '0x3c', '0x62']
```

  </div>
</details>

  <span class="bubble">"케이블이 하나 빠져 있었어요"</span>
</div>

    <span class="label">대응</span>
    <span class="value">케이블을 뺐다가 다시 꽂고, 모든 장치가 I2C0 버스(SDA=GP8, SCL=GP9)에 연결되었는지 확인하세요</span>
  </div>
  <div class="trouble-card">

    <span class="label">대응</span>
    <span class="value">센서를 하나씩 뺐다가 꽂으면서 어떤 장치가 어떤 주소인지 확인하세요</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>이 시점에 순회하며 확인:</strong></p>
  <ol>
    <li>I2C 스캔에서 0x38, 0x3C, 0x62 세 개가 모두 보이는가</li>
    <li>케이블이 단단히 꽂혀 있는가 (느슨한 연결은 간헐적 오류의 원인)</li>
    <li>SCD41 센서의 방향이 올바른가 (센서면이 위를 향하도록)</li>
  </ol>
</div>

<div class="teaching-tip">
  <p>I2C 스캔 코드를 먼저 실행하는 것이 핵심입니다. 하드웨어 연결이 확인되지 않은 상태에서 복잡한 코드를 실행하면 디버깅이 어렵습니다. "연결 먼저, 코드는 그 다음"이라는 원칙을 강조하세요.</p>
</div>

<details class="student-content">
  <summary>학생 교재 — 프롬프트 예시</summary>
  <div class="content">

```
Pico 2 WH에 Grove SCD41 CO2 센서와 DHT20 온습도 센서를 I2C0(SDA=GP8, SCL=GP9)에,
OLED SSD1306도 같은 I2C에 연결했어.

CO2 농도, 온도, 습도를 OLED 대시보드에 실시간 표시하고,
CO2가 1000ppm 넘으면 'VENTILATE!' 경고를 표시하는 MicroPython 코드 만들어줘.

조건:
- SCD41 I2C 주소는 0x62
- 측정 시작 명령: bytes([0x21, 0xB1])
- 5초 대기 후 읽기 명령: bytes([0xEC, 0x05])
- 9바이트 읽기
- CO2 = (data[0]<<8)|data[1]
- DHT20 주소 0x38
- OLED 주소 0x3C, ssd1306 라이브러리 사용
```

  </div>
</details>

    <span class="label">대응</span>

## 정리 (10분)

<div class="teacher-script">
  <p>"자, 환기 실험 결과를 한 팀씩 발표해 봅시다. 창문 닫았을 때 CO2가 가장 높았던 팀은?"</p>
  <p>"창문 열고 나서 CO2가 600ppm 이하로 떨어지는 데 몇 분 걸렸나요?"</p>
  <p>"'20분마다 5분 환기'라는 규칙이 있는데, 오늘 데이터를 보면 이게 적절한 것 같나요?"</p>
  <p>"오늘 여러분이 한 일을 정리하면 이렇습니다. AI에게 프롬프트로 <strong>세 개 센서를 제어하는 코드</strong>를 요청했고, 그 코드를 검증하고 수정해서 돌렸고, 그걸로 <strong>실제 교실 환경을 모니터링</strong>했습니다."</p>
  <p>"1막에서 센서 하나 읽는 코드를 직접 작성하는 데 한 시간 걸렸는데, 오늘은 바이브 코딩으로 센서 세 개짜리 대시보드를 50분 안에 만들었어요. 이것이 2막의 힘입니다."</p>
  <p>"다음 시간에는 <strong>심박 센서</strong>를 씁니다. 여러분의 심장 박동을 OLED에 파형과 BPM으로 표시하는 대시보드를 바이브 코딩으로 만들어 볼 거예요."</p>
</div>

<div class="teaching-tip">
  <p>팀별 데이터를 칠판이나 화면에 모아서 비교하면 효과적입니다. "같은 교실인데 팀마다 CO2가 다른 이유는?" (센서 위치, 주변 인원수 차이)로 이어가면 과학적 사고를 자연스럽게 이끌 수 있습니다.</p>
</div>

## 차별화 전략

<div class="differentiation">
  <div class="fast-learner">
    <p><strong>빠른 학생에게:</strong></p>
    <ul>
      <li>환기 실험 CO2 감소 곡선의 기울기를 분석해 보세요 — 선형인가요, 비선형인가요?</li>
      <li>I2C 스캔 코드로 다른 팀의 연결 문제를 디버깅해 주세요</li>
      <li>교실 인원수와 CO2 증가 속도의 관계를 예측하는 실험을 설계해 보세요</li>
      <li>도전 코드(CO2 히스토리 그래프)를 실행해 보세요</li>
    </ul>
  </div>
  <div class="slow-learner">
    <p><strong>느린 학생에게:</strong></p>
    <ul>
      <li>학생 교재의 기본 코드 탭을 그대로 복사하여 실행하기</li>
      <li>I2C 스캔은 교사가 함께 진행하고, 세 주소가 나오는 것을 확인하기</li>
      <li>환기 실험에 참여하면서 OLED 화면의 CO2 숫자 변화를 관찰하고 기록하기</li>
    </ul>
  </div>
</div>
