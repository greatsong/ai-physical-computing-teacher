## 준비물 체크리스트

<div class="prep-checklist">
  <h5>하드웨어 (팀별 1세트)</h5>
  <ul>
    <li>Raspberry Pi Pico 2 WH + Grove Shield</li>
    <li>초음파 거리 센서 (Ultrasonic Ranger v2) — D16 포트</li>
    <li>부저 (Buzzer) — D18 포트</li>
    <li>LED Bar — D20 포트</li>
    <li>OLED 디스플레이 (SSD1306) — I2C 포트</li>
    <li>Grove 케이블 4개</li>
    <li>USB 데이터 케이블</li>
  </ul>
  <h5>소프트웨어</h5>
  <ul>
    <li>Thonny IDE (MicroPython 인터프리터)</li>
    <li>AI 챗봇 접속 가능한 브라우저 (Claude, ChatGPT 등)</li>
  </ul>
  <h5>수업 전 확인</h5>
  <ul>
    <li>4개 장치를 한 세트로 미리 연결하여 테스트 — 포트 충돌 없이 동작하는지</li>
    <li>부저 볼륨이 교실에 과도하지 않은지 확인 — 필요시 duty 값 낮추기</li>
    <li>자 또는 30cm 눈금자 준비 (거리 측정 테스트용)</li>
    <li>거리별 알림 기록표 인쇄</li>
  </ul>
</div>

<div class="student-voice">

  <p>"그런데 이번엔 장치가 4개입니다. 초음파 센서, 부저, LED Bar, OLED — 역대 최다 장치 조합이에요. 입력 1개로 출력 3개를 동시에 제어합니다. 바이브 코딩이 없었다면 상당히 어려웠을 텐데, AI에게 시키면 어떨까요?"</p>

<div class="teacher-script">
  <p>"오늘은 연결할 장치가 4개라서 가장 복잡합니다. 스크린의 연결표를 보면서 하나씩 꽂아보세요."</p>
  <p>"초음파 센서는 <strong>D16</strong>, 부저는 <strong>D18</strong>, LED Bar는 <strong>D20</strong>, OLED는 <strong>I2C</strong> 포트입니다. 디지털 포트 3개와 I2C 포트 1개 — 총 4개의 케이블을 연결합니다."</p>
  <p>"다 연결했으면 학생 교재의 가상 연결 연습을 해보세요. 올바르게 꽂았는지 확인할 수 있어요."</p>

<div class="student-voice">
  <span class="bubble">"케이블이 4개나 되니까 헷갈려요"</span>

  <span class="bubble">"OLED는 전에도 I2C에 꽂았으니까 알겠어요"</span>

<div class="trouble-cards">

    <span class="label">원인</span>

    <span class="label">대응</span>
    <span class="value">Shield 보드에 인쇄된 포트 번호를 함께 확인하세요. D16, D18, D20은 보드 한쪽에 나란히 있습니다</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"OLED가 안 켜져요"</div>
    <span class="label">원인</span>
    <span class="value">I2C 포트가 아닌 디지털 포트에 연결했거나, 케이블 접촉 불량</span>
    <span class="label">대응</span>
    <span class="value">I2C라고 적힌 포트에 연결했는지 재확인. 케이블을 뺐다 다시 꽂아 보세요</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>이 시점에 순회하며 확인:</strong></p>
  <ol>
    <li>4개 장치가 모두 <strong>올바른 포트</strong>에 연결되어 있는가 (D16, D18, D20, I2C)</li>
    <li>케이블이 단단히 끼워져 있는가 (헐렁하면 간헐적 오류 발생)</li>
    <li>Thonny에서 Pico 인식 확인</li>
  </ol>
</div>

<div class="teaching-tip">
  <p>이번 차시의 연결이 가장 복잡합니다. <strong>2&#126;3분 더 여유</strong>를 주세요. 빠른 학생에게 옆 팀을 도와달라고 부탁하면 효과적입니다. 포트 번호를 칠판에 크게 적어두면 도움이 됩니다.</p>
</div>

<div class="step-header">
  <span class="step-num">2</span>
  <span class="step-title">바이브 코딩으로 알림 시스템 코드 생성</span>
  <span class="step-time">10분</span>
</div>

<div class="teacher-script">
  <p>"자, 이제 AI에게 코드를 시켜봅시다. 학생 교재의 '바이브 코딩 프롬프트' 탭을 열어보세요. 프롬프트가 꽤 길죠? 장치가 4개니까 AI에게 알려줘야 할 정보도 많아요."</p>
  <p>"프롬프트를 그대로 사용해도 좋고, 자기만의 임계값을 넣어서 수정해도 좋습니다. AI가 코드를 생성하면, Thonny에 붙여넣고 실행해보세요."</p>
  <p>"실행하면 센서 앞에 손을 대보세요. 가까이 가면 부저가 울리고, LED가 늘어나고, OLED에 거리가 표시될 거예요."</p>
</div>

<div class="student-voice">
  <span class="bubble">"와, 손 가까이 하니까 삐삐삐 소리 나요!"</span>

  <span class="bubble">"LED가 점점 늘어나요"</span>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"부저가 안 울려요"</div>
    <span class="label">원인</span>
    <span class="value">부저 핀 번호가 코드와 맞지 않거나, D18 포트 연결 불량</span>
    <span class="label">대응</span>
    <span class="value">코드에서 Pin(18)인지 확인. 부저 케이블을 뺐다 다시 꽂으세요</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"거리가 이상하게 나와요" (0cm 또는 999cm)</div>
    <span class="label">원인</span>
    <span class="value">초음파 센서 앞에 장애물이 너무 가깝거나 없는 경우, 또는 타이밍 오류</span>
    <span class="label">대응</span>
    <span class="value">센서 앞 5&#126;50cm 범위에 평평한 물체(책 등)를 놓고 테스트. 코드에 타임아웃 처리가 있는지 확인</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"부저가 너무 시끄러워요!"</div>

    <span class="label">대응</span>

  </div>
</div>

<div class="teaching-tip">
  <p><strong>부저 소음 관리</strong>가 이 차시의 핵심 관리 포인트입니다. 15팀이 동시에 부저를 울리면 교실이 매우 시끄러워집니다. 미리 "테스트할 때는 손으로 부저를 반쯤 가리세요"라고 안내하거나, duty 값을 8192 이하로 제한하세요.</p>
</div>

## 정리 (10분)

<div class="teacher-script">
  <p>"오늘 여러분이 만든 시스템을 다시 생각해볼까요?"</p>
  <p>"초음파 센서가 거리를 <strong>입력</strong>하면, 코드가 거리를 판단해서 <strong>처리</strong>하고, 부저 + LED + OLED 세 가지로 <strong>출력</strong>했어요. 입력 → 처리 → 출력. 이게 피지컬 컴퓨팅의 핵심 구조예요."</p>
  <p>"자동차 후방 센서도, 로봇 청소기도, 공장의 안전 시스템도 전부 이 구조입니다. 여러분은 오늘 그 원리를 직접 만들어본 거예요."</p>
  <p>"그리고 하나 더. 임계값만 바꿨는데 주차 보조에서 복도 안전 시스템으로 용도가 바뀌었죠? <strong>기술은 같아도 설계가 다르면 다른 제품</strong>이 됩니다. 이것이 엔지니어링이에요."</p>
  <p>"다음 시간에는 Pico를 내려놓고, 컴퓨터 Python을 씁니다. 지금까지 센서로 모은 데이터를 정리하고 그래프를 그려봅니다. 데이터 과학의 세계로 넘어갑니다."</p>
</div>

<div class="teaching-tip">
  <p><strong>"입력 → 처리 → 출력"</strong> 구조를 칠판에 그려주면 효과적입니다. 초음파(입력) → if/elif 판단(처리) → 부저+LED+OLED(출력)를 화살표로 연결하세요. 다음 시간은 Pico가 아니라 PC Python이라는 점을 명확히 예고하세요.</p>
</div>

    <p><strong>빠른 학생에게:</strong></p>
    <ul>
      <li>거리별 경고를 4단계가 아닌 5단계로 세분화하고, 각 임계값의 근거를 설명해보기</li>
      <li>PWM 주파수를 200Hz&#126;4000Hz 범위에서 바꿔가며 "사람이 가장 주목하는 주파수"를 실험으로 찾아보기</li>
      <li>시각장애인 보행 보조, 교실 혼잡 감지 등 사회적 응용 시나리오를 설계해보기</li>
      <li>다른 팀의 시스템 디버깅을 도와주며 "분할 정복" 전략 공유하기</li>
    </ul>
  </div>
  <div class="slow-learner">
    <p><strong>느린 학생에게:</strong></p>
    <ul>
      <li>포트 연결을 교사가 직접 확인해주고, 학생 교재의 가상 연결 연습부터 시작</li>
      <li>기본 코드 탭의 코드를 복사/붙여넣기로 실행한 뒤, 손만 가까이/멀리 움직이며 체험</li>
      <li>임계값 숫자 하나만 바꿔보는 작은 실험 (예: 10 → 15)</li>
    </ul>
  </div>
</div>
