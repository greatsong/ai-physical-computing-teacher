## 도입 (5분)

<div class="teacher-script">
  <p>"여러분, 1막에서 센서 코드를 직접 작성했었죠? DHT20 온도 읽는 코드, 빛 센서 그래프 그리는 코드... 한 줄씩 타이핑하면서 만들었잖아요."</p>
  <p>"오늘부터 2막입니다. 2막의 키워드는 <strong>바이브 코딩(Vibe Coding)</strong>이에요."</p>
  <p>"바이브 코딩이 뭐냐면, AI에게 '이런 코드 만들어줘'라고 <strong>말로 설명</strong>하면, AI가 코드를 작성해주는 방식이에요. 2025년부터 전 세계 개발자들이 실제로 이렇게 일하고 있습니다."</p>
</div>

<div class="student-voice">
  <span class="bubble">"진짜 말로만 하면 코드가 나와요?"</span>
  <span class="bubble">"그럼 코딩 안 배워도 되는 거 아니에요?"</span>
  <span class="bubble">"1막에서 그 고생을 왜 한 거예요?"</span>
</div>

<div class="teacher-script">
  <p>"좋은 질문이에요. '코딩 안 배워도 되는 거 아니냐'라는 질문, 오늘 수업 끝나고 다시 물어볼게요. 그때 대답이 달라져 있을 겁니다."</p>
  <p>"오늘 수업이 끝나면, 여러분은 AI에게 프롬프트를 보내서 코드를 받고, 그 코드를 <strong>읽고, 검증하고, 수정</strong>하는 경험을 하게 됩니다."</p>

</div>

## 전개 (40분)

<div class="step-header">
  <span class="step-num">1</span>
  <span class="step-title">첫 번째 바이브 코딩</span>
  <span class="step-time">15분</span>
</div>

<div class="teacher-script">
  <p>"자, AI 챗봇을 열어보세요. Claude든 ChatGPT든 상관없습니다."</p>
  <p>"이제 AI에게 코드를 시킬 건데요. 아무렇게나 '온도 코드 만들어줘'라고 하면 될까요? 아뇨. <strong>프롬프트</strong>를 잘 써야 합니다."</p>
  <p>"화면에 보이는 프롬프트를 참고해서 AI에게 보내보세요. 그대로 베껴도 좋고, 나만의 방식으로 수정해도 좋습니다."</p>
</div>

<details class="student-content">
  <summary>학생 교재 — 프롬프트 예시</summary>
  <div class="content">

```
Raspberry Pi Pico 2 WH에 Grove Shield를 사용하고 있어.
Grove DHT20 온습도 센서를 I2C1 포트(SDA=GP6, SCL=GP7)에 연결했어.
2초마다 온도와 습도를 측정해서 출력하는 MicroPython 코드를 만들어줘.
```

  </div>
</details>

<div class="teacher-script">
  <p>"AI가 코드를 줬죠? 바로 실행하지 마세요. 먼저 한번 읽어보세요."</p>
  <p>"1차시에서 여러분이 직접 작성한 코드와 비교해 봅시다. 구조가 비슷한가요? AI가 추가한 부분이 있나요? <strong>주석</strong>을 달아줬나요?"</p>
  <p>"좋아요, 이제 Thonny에 붙여넣고 실행해 봅시다."</p>
</div>

<div class="student-voice">
  <span class="bubble">"와, 진짜 코드가 나왔어요!"</span>
  <span class="bubble">"주석이 엄청 많아요"</span>
  <span class="bubble">"실행하니까 바로 온도가 나와요!"</span>
  <span class="bubble">"어? 근데 에러가 나요..."</span>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"AI 코드에 없는 라이브러리를 import 해요"</div>
    <span class="label">원인</span>
    <span class="value">AI가 표준 Python 라이브러리를 사용했거나, 존재하지 않는 DHT20 라이브러리를 가정한 경우</span>
    <span class="label">대응</span>
    <span class="value">"AI가 틀린 거예요. 1차시에서 우리가 쓴 방식처럼 I2C 직접 통신 코드로 바꿔야 합니다. AI에게 '라이브러리 없이 I2C로 직접 통신하는 코드로 다시 만들어줘'라고 요청하세요."</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"핀 번호가 달라요"</div>
    <span class="label">원인</span>
    <span class="value">AI가 다른 보드의 핀 번호를 사용했거나, SDA/SCL을 바꿔 쓴 경우</span>
    <span class="label">대응</span>
    <span class="value">"프롬프트에 핀 번호를 명시했는데도 틀렸나요? AI는 항상 맞는 게 아닙니다. SDA=GP6, SCL=GP7로 직접 수정하세요."</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"I2C 주소가 달라요 (0x38이 아닌 다른 주소)"</div>
    <span class="label">원인</span>
    <span class="value">AI가 DHT20의 I2C 주소를 잘못 알고 있는 경우</span>
    <span class="label">대응</span>
    <span class="value">"1차시에서 우리가 확인한 주소는 0x38이었죠? AI 코드의 주소를 0x38로 고치세요. 이게 바로 직접 코딩 경험이 빛나는 순간입니다."</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>이 시점에 순회하며 확인:</strong></p>
  <ol>
    <li>AI 챗봇에 프롬프트를 보냈는가 (복사/붙여넣기도 OK)</li>
    <li>AI 코드를 바로 실행하지 않고 먼저 <strong>읽어보았는가</strong></li>
    <li>1차시 코드와 비교해 보았는가</li>
    <li>에러가 나는 팀이 있다면, AI에게 에러 메시지를 다시 보내도록 안내</li>
  </ol>
</div>

<div class="teaching-tip">
  <p>이 단계에서 AI 코드에 오류가 있는 것이 <strong>오히려 좋습니다</strong>. 오류가 없으면 "AI 만능"으로 느끼고, 오류가 있으면 "검증이 필요하다"는 교훈을 얻습니다. AI가 DHT20 라이브러리를 import 하는 코드를 내놓는 경우가 많은데, 이것은 "AI가 하드웨어 맥락을 모른다"는 좋은 사례입니다.</p>
</div>

<div class="step-header">
  <span class="step-num">2</span>
  <span class="step-title">프롬프트 업그레이드</span>
  <span class="step-time">15분</span>
</div>

<div class="teacher-script">
  <p>"기본 코드가 돌아가는 팀, 손 들어보세요. 좋습니다."</p>
  <p>"이제 프롬프트를 <strong>업그레이드</strong>해 봅시다. AI에게 기능을 추가해 달라고 요청할 거예요."</p>
  <p>"교재에 있는 프롬프트를 보세요. 온도 28도 이상이면 '더워요!' 출력, 습도 70% 이상이면 '습해요!' 출력, 측정 횟수 표시, 10번 측정 후 자동 종료 — 이 네 가지를 AI에게 요청합니다."</p>
  <p>"여기서 중요한 건, <strong>한 번에 완벽한 코드를 기대하지 않는 것</strong>입니다. AI가 준 코드가 의도대로 안 되는 부분이 있으면 직접 수정해 보세요."</p>
</div>

<details class="student-content">
  <summary>학생 교재 — 프롬프트 업그레이드</summary>
  <div class="content">

```
위 코드에 다음 기능을 추가해줘:
1. 온도가 28도 이상이면 "더워요!"라고 출력
2. 습도가 70% 이상이면 "습해요!"라고 출력
3. 측정 횟수를 세서 "N번째 측정" 형태로 표시
4. 10번 측정하면 자동으로 종료
```

  </div>
</details>

<div class="student-voice">
  <span class="bubble">"오, 기능이 추가됐어요!"</span>
  <span class="bubble">"근데 '더워요'가 안 나와요, 교실이 안 더우니까..."</span>
  <span class="bubble">"28도를 25도로 바꾸면 되지 않아요?"</span>
  <span class="bubble">"센서에 손을 대니까 온도 올라가서 '더워요' 떠요!"</span>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"10번 측정 후에 종료가 안 돼요"</div>
    <span class="label">원인</span>
    <span class="value">AI가 while True를 그대로 뒀거나, break 조건을 잘못 넣은 경우</span>
    <span class="label">대응</span>
    <span class="value">"while 루프 안에서 count가 10이 되면 break 하도록 고쳐보세요. 1막에서 while과 if를 배웠으니 직접 수정할 수 있어요."</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>순회 시 확인:</strong></p>
  <ol>
    <li>프롬프트를 단계적으로 요청하고 있는가 (한 번에 모든 걸 요구하지 않는가)</li>
    <li>AI 코드의 오류를 직접 수정하려고 시도하는가</li>
    <li>수정한 부분을 짝과 공유하는가</li>
  </ol>
</div>

<div class="teaching-tip">
  <p>이 단계에서 학생이 AI 코드를 직접 수정하는 경험이 매우 중요합니다. "AI한테 다시 물어보면 되잖아요"라고 하는 학생에게는 "그것도 방법이지만, 직접 고치는 게 더 빠를 때도 있어요. 1차시에서 배운 if 문이면 충분해요"라고 안내하세요.</p>
</div>

  <span class="step-num">3</span>
  <span class="step-title">나만의 프롬프트 도전</span>

<div class="teacher-script">

  <p>"A는 쾌적도 판정기 — 온도와 습도 조합으로 쾌적/보통/불쾌를 판정합니다."</p>

## 정리 (5분)

<div class="teacher-script">
  <p>"수업 처음에 질문했죠? '코딩 안 배워도 되는 거 아니에요?'라고. 지금 답이 바뀌었나요?"</p>
  <p>"오늘 AI가 준 코드에 오류가 있었던 팀, 손 들어보세요. 꽤 많죠? AI가 핀 번호를 틀리거나, 없는 라이브러리를 쓰거나, I2C 주소를 잘못 넣기도 했어요."</p>
  <p>"여러분이 그 오류를 발견할 수 있었던 이유가 뭘까요? <strong>1막에서 직접 코딩해 봤기 때문</strong>이에요. I2C가 뭔지, 핀 번호가 어떤 건지 알고 있으니까, AI가 틀렸을 때 '이거 아닌데?'라고 알 수 있는 거예요."</p>
  <p>"AI는 <strong>패턴 조합은 빠르지만</strong>, 여러분의 <strong>하드웨어를 직접 보고 있지는 않습니다</strong>. 그래서 검증하는 사람이 필요하고, 그 사람이 바로 여러분이에요."</p>
  <p>"다음 시간에는 바이브 코딩으로 <strong>CO2 센서 + 온습도 센서 + OLED 화면</strong>을 조합한 환경 모니터링 대시보드를 만듭니다. 직접 코딩이라면 한 시간 넘게 걸릴 시스템을, AI에게 시켜서 빠르게 완성해 봅시다."</p>
</div>

<div class="teaching-tip">
  <p>"직접 코딩 vs 바이브 코딩"의 관계를 <strong>대립이 아니라 상호보완</strong>으로 정리하세요. "직접 코딩을 할 줄 알아야 AI가 틀렸을 때 고칠 수 있다. 바이브 코딩은 속도를 주고, 직접 코딩은 검증 능력을 준다." 이 메시지가 2막 전체를 관통하는 주제입니다.</p>
</div>

## 차별화 전략

<div class="differentiation">
  <div class="fast-learner">
    <p><strong>빠른 학생에게:</strong></p>
    <ul>
      <li>같은 프롬프트를 Claude와 ChatGPT에 각각 보내서 결과를 비교해 보세요</li>
      <li>프롬프트를 세 번 구체화하면서 품질 변화를 스프레드시트로 정리해 보세요</li>
      <li>AI 코드와 1차시 직접 작성 코드를 줄 단위로 비교하여 "프롬프트 작성 가이드"를 만들어 보세요</li>
    </ul>
  </div>
  <div class="slow-learner">
    <p><strong>느린 학생에게:</strong></p>
    <ul>
      <li>교사가 미리 준비한 프롬프트 예시를 그대로 복사하여 AI에게 보내기</li>
      <li>AI 코드를 그대로 붙여넣고 실행하는 것부터 시작</li>
      <li>오류가 나면 교사가 "이 줄의 숫자를 0x38로 바꿔보세요"처럼 구체적으로 안내</li>
    </ul>
  </div>
</div>
