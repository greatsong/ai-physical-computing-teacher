## 준비물 체크리스트

<div class="prep-checklist">
  <h5>하드웨어 (팀별)</h5>
  <ul>
    <li>팀 프로젝트 결과물 (대시보드가 동작하는 상태)</li>
    <li>OLED + Pico + 센서 세트 (라이브 시연용)</li>
    <li>USB 케이블 + 교실 프로젝터/모니터 연결</li>
  </ul>
  <h5>발표 자료</h5>
  <ul>
    <li>슬라이드, 인쇄물, 또는 대시보드 직접 시연</li>
    <li>ML 결과 스크린샷 또는 Teachable Machine 실시간 시연</li>
  </ul>
  <h5>수업 전 확인</h5>
  <ul>
    <li>발표 순서 사전 결정 (추첨 또는 자원)</li>
    <li>타이머 준비 (팀당 5분 발표 + 2분 질의응답)</li>
    <li>피드백 양식 인쇄 또는 디지털 공유 (팀 수만큼 준비)</li>
    <li>프로젝터/모니터가 OLED 화면을 확대해 보여줄 수 있는가 (문서 카메라 활용 가능)</li>
    <li>각 팀이 발표 전 1분간 셋업할 시간 확보</li>
  </ul>
</div>

## 도입 (5분)

<div class="teacher-script">
  <p>"오늘은 여러분이 3주 동안 만든 프로젝트를 <strong>발표하는 날</strong>입니다!"</p>
  <p>"엔지니어와 데이터 과학자에게 가장 중요한 능력 중 하나가 뭔지 아세요? <strong>자기가 만든 것을 설명하는 능력</strong>이에요. 아무리 좋은 대시보드를 만들어도, 설명을 못 하면 아무도 쓰지 않거든요."</p>
  <p>"발표 구조를 다시 한번 확인합니다. 5단계예요 — <strong>문제 발견, 센서/대시보드 시연, 데이터 분석, ML 결과, 의미와 시사점</strong>. 각 팀 5분 발표, 2분 질의응답입니다."</p>
  <p>"발표를 듣는 팀은 <strong>피드백 양식</strong>을 작성해 주세요. 대시보드 디자인, 데이터 신뢰성, ML 활용도, 전달력, 실용성 — 이 5개 항목에 1&#126;5점을 매기고, 한 줄 코멘트를 적어주세요."</p>
</div>

<div class="student-voice">
  <span class="bubble">"긴장되는데&#126;"</span>
  <span class="bubble">"시연이 잘 될까?"</span>
  <span class="bubble">"피드백 양식 받았어요"</span>
</div>

<div class="teaching-tip">
  <p>"긴장되는 건 당연하다"고 안심시켜 주세요. "라이브 시연 중에 에러가 나도 괜찮아요. 에러가 났을 때 어떻게 대처하는지가 진짜 실력이에요"라는 메시지가 학생들의 부담을 줄여줍니다. 긴장하는 팀원을 격려하는 행동도 평가에 포함된다고 안내하세요.</p>
</div>

## 전개 (35분)

<div class="step-header">
  <span class="step-num">1</span>
  <span class="step-title">발표 최종 준비</span>
  <span class="step-time">5분</span>
</div>

<div class="teacher-script">
  <p>"5분 드릴 테니 팀별로 마지막 점검하세요. 대시보드가 동작하는지 확인하고, 발표 순서와 각자 맡은 부분을 정리하세요."</p>
  <p>"발표 팁 하나 — 슬라이드보다 <strong>대시보드를 실제로 시연</strong>하는 게 훨씬 인상적이에요. OLED에 데이터가 나오는 모습, PC 그래프가 움직이는 모습, Teachable Machine이 소리를 맞추는 모습을 직접 보여주세요."</p>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"대시보드가 갑자기 안 돼요!"</div>
    <span class="label">원인</span>
    <span class="value">USB 연결 불량, 배터리 부족, 코드 실행 안 됨</span>
    <span class="label">대응</span>
    <span class="value">USB를 다시 꽂고 Thonny에서 코드를 재실행하세요. 최악의 경우, 이전에 찍어둔 스크린샷/동영상으로 시연을 대체하세요</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>순회 시 확인:</strong></p>
  <ol>
    <li>모든 팀이 시연 준비가 되었는가</li>
    <li>피드백 양식을 모든 학생이 갖고 있는가</li>
    <li>긴장하는 학생에게 격려의 한마디</li>
  </ol>
</div>

<div class="step-header">
  <span class="step-num">2</span>
  <span class="step-title">팀별 발표 + 질의응답</span>
  <span class="step-time">25분</span>
</div>

<div class="teacher-script">
  <p>"첫 번째 팀, 나와주세요! 5분입니다. 타이머 시작!"</p>
  <p>(각 팀 발표 후) "질문 있는 팀?"</p>
  <p>(질의응답 후) "좋아요, 피드백 양식 작성해 주세요. 다음 팀 준비!"</p>
</div>

<div class="student-voice">
  <span class="bubble">"저희 팀은 교실 환경 측정 프로젝트입니다"</span>
  <span class="bubble">"지금 OLED에 실시간 온도가 나오고 있어요"</span>
  <span class="bubble">"ML로 환기 필요 상태를 87% 정확도로 감지했어요"</span>
  <span class="bubble">"데이터가 왜 그렇게 나왔어요?"</span>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">발표 중 시연 실패</div>
    <span class="label">대응</span>
    <span class="value">"라이브 시연은 원래 긴장되죠. 잘 돌아갈 때 어떤 모습이었는지 구두로 설명해 볼까요?"라고 자연스럽게 이어가세요. 실패를 학습 기회로 전환하는 것이 중요합니다</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">시간이 초과됨</div>
    <span class="label">대응</span>
    <span class="value">"30초 남았습니다" 신호를 주세요. 4분에 한 번, 4분 30초에 한 번. 5분이 넘어가면 부드럽게 마무리를 유도하세요</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>발표 진행 시 확인:</strong></p>
  <ol>
    <li>발표 5단계 구조(문제→시연→데이터→ML→시사점)를 따르고 있는가</li>
    <li>데이터 기반 주장을 하고 있는가 (숫자와 그래프로 뒷받침)</li>
    <li>청중이 피드백 양식을 작성하고 있는가</li>
    <li>질의응답이 건설적으로 이루어지는가</li>
  </ol>
</div>

<div class="teaching-tip">
  <p>발표 사이에 <strong>30초 전환 시간</strong>을 주세요. 다음 팀이 USB를 연결하고 코드를 실행하는 시간이 필요합니다. 질의응답이 부족하면 교사가 "대시보드에서 가장 인상적인 부분이 뭐였나요?", "ML 정확도를 더 높이려면 어떻게 해야 할까요?" 같은 질문으로 유도하세요. 국어 시간의 논증 구조(주장-근거-반박)와 연결하면 교과 통합 효과가 있습니다.</p>
</div>

<div class="step-header">
  <span class="step-num">3</span>
  <span class="step-title">피드백 집계</span>
  <span class="step-time">5분</span>
</div>

<div class="teacher-script">
  <p>"모든 발표가 끝났습니다! 수고했어요."</p>
  <p>"이제 피드백을 집계할 거예요. 교재의 피드백 집계 코드를 열어 보세요. 여러분 팀이 받은 점수를 입력하면, 항목별 평균과 강점/보완점이 자동으로 나옵니다."</p>
  <p>"다른 팀에서 받은 피드백 양식을 모아서, 5개 항목의 점수를 코드에 넣어 보세요."</p>
</div>

<div class="student-voice">
  <span class="bubble">"우리 팀 대시보드 디자인이 4.2점이래요!"</span>
  <span class="bubble">"ML 활용도가 좀 낮네&#126;"</span>
  <span class="bubble">"강점이 데이터 신뢰성이래요"</span>
</div>

<div class="teaching-tip">
  <p>피드백 집계 시간이 부족하면, 코드 실행은 숙제로 돌리고 구두로 주요 피드백만 공유하세요. "비판은 공격이 아니라 개선의 기회"라는 메시지를 다시 한번 강조하세요. "ML 활용도 점수가 낮다"는 피드백은 "ML을 더 의미 있게 연결할 기회가 있다"는 뜻이라고 재해석해 주세요.</p>
</div>

## 정리 (10분)

<div class="teacher-script">
  <p>"오늘 여러분이 한 일의 의미를 생각해볼까요?"</p>
  <p>"<strong>문제 발견→센서 선택→대시보드 제작→ML 학습→발표→피드백</strong> — 이 전체 과정은 IT 프로젝트의 실제 개발 사이클과 동일해요."</p>
  <p>"스타트업이 새로운 서비스를 만들 때, 데이터 과학자가 분석 결과를 보고할 때 — 모두 '문제→프로토타입→데이터→발표→피드백'을 반복합니다. 여러분은 이 사이클을 4차시에 걸쳐 직접 경험한 거예요."</p>
  <p>"그리고 피드백도 결국 <strong>데이터</strong>예요. 숫자로 모으고, 평균을 내고, 패턴을 찾는 것 — 센서 데이터를 다루는 것과 같은 원리로 피드백을 분석했어요."</p>
  <p>"다음 시간은 코스 A의 <strong>마지막 차시</strong>입니다. 15주 동안의 여정을 되돌아보고, 얼마나 성장했는지 확인해볼 거예요."</p>
</div>

<div class="teaching-tip">
  <p>모든 팀의 노력을 인정해 주세요. "이 팀은 대시보드 디자인이 뛰어나고, 저 팀은 데이터 분석이 깊었고, 이 팀은 ML 활용이 창의적이었다" — 각 팀만의 강점을 한 가지씩 꼭 언급하세요. 완성도가 낮은 팀에게는 "도전한 것 자체가 대단한 것"이라고 격려하고, 구체적으로 잘한 부분을 찾아주세요.</p>
</div>

## 차별화 전략

<div class="differentiation">
  <div class="fast-learner">
    <p><strong>빠른 학생에게:</strong></p>
    <ul>
      <li>"문제→데이터→분석→제안" 논리적 발표 구성에 도전하세요</li>
      <li>라이브 시연 중 예상치 못한 수치를 즉석으로 해석해 보세요</li>
      <li>다른 팀에 구체적이고 실행 가능한 개선안을 제시해 보세요</li>
      <li>피드백 집계 도전 코드(전체 학급 대시보드)를 실행하여 학급 전체 강점을 시각화하세요</li>
      <li>긴장하는 팀원을 격려해 주세요</li>
    </ul>
  </div>
  <div class="slow-learner">
    <p><strong>느린 학생에게:</strong></p>
    <ul>
      <li>발표에서 자신이 맡은 한 가지 부분만 잘 설명하면 됩니다</li>
      <li>피드백 양식에 항목별 차등 점수를 매기는 연습을 하세요</li>
      <li>피드백 코멘트는 "&#126;가 좋았다", "&#126;을 더 하면 좋겠다" 형식으로 작성하세요</li>
      <li>발표가 어려우면 대시보드 시연만 담당해도 좋습니다</li>
    </ul>
  </div>
</div>
