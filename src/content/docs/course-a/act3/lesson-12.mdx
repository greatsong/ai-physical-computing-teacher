## 준비물 체크리스트

<div class="prep-checklist">
  <h5>하드웨어 (팀별 1세트)</h5>
  <ul>
    <li>Raspberry Pi Pico 2 WH + Grove Shield</li>
    <li>팀별 선택 센서 (11차시 계획서 기준 1&#126;3개)</li>
    <li>OLED 디스플레이 SSD1306 (I2C)</li>
    <li>Grove 케이블 (센서 수 + OLED용)</li>
    <li>USB 데이터 케이블</li>
  </ul>
  <h5>소프트웨어</h5>
  <ul>
    <li>Thonny IDE (ssd1306 라이브러리 설치 완료)</li>
    <li>PC Python 3 + pyserial + matplotlib (PC 대시보드용)</li>
  </ul>
  <h5>수업 전 확인</h5>
  <ul>
    <li>ssd1306 라이브러리가 Pico에 설치되어 있는가 — <code>import ssd1306</code>로 확인</li>
    <li>각 팀의 센서가 해당 포트에서 동작하는가 (최소 1세트 사전 테스트)</li>
    <li>OLED 화면이 켜지는지 확인 (I2C 주소 0x3C)</li>
    <li>Thonny와 PC Python의 serial 포트 충돌 안내 준비 (Thonny 종료 후 PC 코드 실행)</li>
  </ul>
</div>

<div class="teacher-script">
  <p>"지난 시간에 프로젝트 계획서를 만들었죠? 오늘은 그 설계를 <strong>실제로 만드는 날</strong>입니다."</p>
  <p>"오늘의 목표는 두 가지예요. 첫째, <strong>OLED 대시보드</strong>에 팀 센서 데이터를 실시간으로 표시하기. 둘째, <strong>PC 대시보드</strong>에서 같은 데이터를 그래프로 보기."</p>
  <p>"그런데 OLED 대시보드 코드를 처음부터 직접 쓰면 시간이 부족하겠죠? 11차시에 세운 바이브 코딩 전략을 기억하세요. AI에게 대시보드 코드를 요청하고, 우리 센서에 맞게 <strong>수정하고 디버깅</strong>하는 것이 오늘의 핵심 기술입니다."</p>
</div>

<div class="student-voice">

  <span class="bubble">"PC에서도 보이게 하고 싶어요!"</span>
</div>

<div class="teaching-tip">

</div>

<div class="teacher-script">
  <p>"먼저 AI에게 대시보드 코드를 요청합니다. 프롬프트 작성 요령을 알려줄게요."</p>
  <p>"<strong>다섯 가지</strong>를 반드시 포함하세요. 하드웨어 이름, 센서 이름과 포트, OLED 사양, 화면 레이아웃, 갱신 주기. 이 다섯 가지가 빠지면 AI가 엉뚱한 코드를 줍니다."</p>
  <p>"예시 프롬프트를 보여드릴게요. (스크린에 예시를 띄움) 이걸 참고해서 <strong>우리 팀 센서에 맞게</strong> 수정하세요."</p>
</div>

<div class="student-voice">
  <span class="bubble">"프롬프트 다 썼어요!"</span>
  <span class="bubble">"코드가 50줄이나 나왔어요"</span>
  <span class="bubble">"이거 바로 돌려볼까요?"</span>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"AI가 다른 보드(Arduino) 코드를 줘요"</div>
    <span class="label">원인</span>
    <span class="value">프롬프트에 "MicroPython", "Raspberry Pi Pico 2 WH"가 빠짐</span>
    <span class="label">대응</span>
    <span class="value">프롬프트 첫 줄에 "MicroPython으로 작성해줘. 보드는 Raspberry Pi Pico 2 WH야."를 추가하세요</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"코드가 너무 길어서 뭐가 뭔지 모르겠어요"</div>
    <span class="label">원인</span>
    <span class="value">AI가 과도한 기능을 포함시킴</span>
    <span class="label">대응</span>
    <span class="value">"센서 읽기 + OLED 표시만 해줘. 다른 기능은 빼줘."로 다시 요청하세요</span>
  </div>
</div>

<div class="teaching-tip">
  <p>바이브 코딩 프롬프트 작성을 <strong>급하게 넘기지 마세요</strong>. 프롬프트의 질이 결과 코드의 질을 결정합니다. "AI가 준 코드가 바로 돌아가지 않는 건 정상"이라고 미리 말해두세요. 디버깅 과정 자체가 학습입니다.</p>
</div>

<div class="step-header">
  <span class="step-num">2</span>
  <span class="step-title">OLED 대시보드 구현 및 디버깅</span>
  <span class="step-time">10분</span>
</div>

<div class="teacher-script">
  <p>"AI가 준 코드를 Thonny에 붙여넣고 실행해 보세요. 아마 바로 돌아가지 않을 거예요. 그게 정상입니다."</p>
  <p>"에러가 나면 에러 메시지를 <strong>읽어</strong> 보세요. 줄 번호가 나오니까 그 줄을 먼저 확인하세요."</p>
  <p>"코드 검증 체크리스트를 하나씩 확인합니다. 센서 포트 번호 맞는지, OLED I2C 주소가 0x3C인지, 화면에 글자가 겹치지 않는지, 갱신 주기가 적절한지."</p>
</div>

<div class="student-voice">
  <span class="bubble">"OSError 나요!"</span>
  <span class="bubble">"OLED에 아무것도 안 나와요"</span>
  <span class="bubble">"글자가 겹쳐요"</span>
  <span class="bubble">"오, 온도 나온다!"</span>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"OSError: [Errno 5] EIO"</div>
    <span class="label">원인</span>
    <span class="value">I2C 통신 실패 — OLED 또는 센서의 주소/연결 문제</span>
    <span class="label">대응</span>
    <span class="value">쉘에서 <code>i2c.scan()</code>을 실행하여 연결된 I2C 주소를 확인하세요. OLED는 0x3C, DHT20은 0x38이 나와야 합니다</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"OLED 화면이 깜빡여요"</div>
    <span class="label">원인</span>
    <span class="value">갱신 주기가 너무 빠르거나, oled.fill(0)과 oled.show() 사이 처리가 느림</span>
    <span class="label">대응</span>
    <span class="value">time.sleep(1)을 time.sleep(2)로 늘리거나, 화면 그리기 코드를 간소화하세요</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"글자가 화면 밖으로 나가요"</div>
    <span class="label">원인</span>
    <span class="value">OLED은 128x64 픽셀, 문자 하나는 8x8. 한 줄에 16자, 총 8줄이 한계</span>
    <span class="label">대응</span>
    <span class="value">텍스트를 줄이거나 약어를 사용하세요. y좌표를 8단위로 배치하면 겹치지 않습니다</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>이 시점에 순회하며 확인:</strong></p>
  <ol>
    <li>OLED에 센서 값이 표시되고 있는가 (아직이면 에러 메시지 함께 확인)</li>
    <li>AI 코드에서 센서 주소/포트를 팀 센서에 맞게 수정했는가</li>
    <li>화면 레이아웃이 가독성 있는가</li>
    <li>2개 이상 센서를 쓰는 팀은 I2C 주소 충돌이 없는가</li>
  </ol>
</div>

<div class="teaching-tip">
  <p>이 단계에서 <strong>가장 흔한 오류는 I2C 주소와 포트 번호</strong>입니다. AI가 생성한 코드는 일반적인 설정을 사용하므로 실제 연결과 다를 수 있습니다. <code>i2c.scan()</code>으로 현재 연결된 장치를 확인하는 습관을 들이세요. 정보 시간에 배운 "모듈화" 개념을 연결하여 "센서 읽기 함수와 화면 그리기 함수를 분리하면 디버깅이 쉬워진다"고 안내할 수 있습니다.</p>
</div>

<div class="step-header">
  <span class="step-num">3</span>
  <span class="step-title">PC 대시보드 연동</span>
  <span class="step-time">10분</span>
</div>

<div class="teacher-script">
  <p>"OLED 대시보드가 동작하는 팀은 다음 단계로 갑니다. Pico에서 <code>print()</code>로 출력한 데이터를 PC에서 받아 그래프로 그리는 <strong>이중 대시보드</strong>를 만들어볼 거예요."</p>
  <p>"중요한 점 — <strong>Thonny를 완전히 종료</strong>한 뒤에 PC Python으로 serial 포트를 열어야 합니다. Thonny가 열려 있으면 포트를 독점해서 PC에서 열 수 없어요."</p>
  <p>"OLED 코드에 CSV 형식 print 출력이 있죠? 이걸 PC에서 받아서 matplotlib으로 그래프를 그립니다."</p>
</div>

<div class="student-voice">
  <span class="bubble">"포트 이름이 뭐예요?"</span>
  <span class="bubble">"COM3? COM4?"</span>
  <span class="bubble">"그래프가 움직여요!"</span>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"serial.SerialException: could not open port"</div>
    <span class="label">원인</span>
    <span class="value">Thonny가 serial 포트를 점유 중</span>
    <span class="label">대응</span>
    <span class="value">Thonny를 완전히 종료(X 버튼)하고 다시 시도하세요. Pico 코드를 main.py로 저장해두면 Thonny 없이도 자동 실행됩니다</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"데이터가 깨져서 나와요"</div>
    <span class="label">원인</span>
    <span class="value">Baud rate 불일치 또는 print 형식이 CSV가 아님</span>
    <span class="label">대응</span>
    <span class="value">Pico와 PC 모두 115200으로 설정, print 출력이 "값1,값2,값3" 형식인지 확인하세요</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>순회 시 확인:</strong></p>
  <ol>
    <li>OLED가 동작 중인 팀은 PC 연동을 시도하고 있는가</li>
    <li>serial 포트 이름을 정확히 찾았는가 (장치 관리자에서 확인)</li>
    <li>아직 OLED도 완성 안 된 팀은 PC 연동을 건너뛰고 OLED에 집중하도록</li>
  </ol>
</div>

<div class="teaching-tip">
  <p>PC 대시보드 연동은 <strong>시간이 부족할 수 있습니다</strong>. OLED 대시보드 완성이 우선이고, PC 연동은 도전 과제로 남겨도 됩니다. deque 버퍼 크기를 60에서 30으로 줄여보는 실험은 빠른 팀에게 제안하세요. "데이터 인터페이스를 사전에 합의하면 통합 오류를 줄일 수 있다"는 점을 강조하면 소프트웨어 공학 개념으로 연결됩니다.</p>
</div>

## 정리 (10분)

<div class="teacher-script">
  <p>"여러분이 오늘 만든 것을 한번 볼까요? OLED에 센서 데이터가 실시간으로 나오고, PC에서는 같은 데이터가 그래프로 그려지고 있어요."</p>
  <p>"이것은 단순한 '센서 읽기'가 아닙니다. 여러분은 방금 <strong>IoT 시스템의 핵심 구조</strong>를 만든 거예요. 센서 계층, 엣지 디바이스, 통신 계층, 대시보드 — 스마트홈의 온도 조절기, 병원의 환자 모니터도 이 구조의 확장이에요."</p>
  <p>"다음 시간에는 <strong>Teachable Machine</strong>으로 ML 모델을 만들어봅니다. 오늘 대시보드에서 수집한 데이터가 AI의 입력이 됩니다."</p>
</div>

<div class="teaching-tip">
  <p>팀마다 완성도가 다를 수 있습니다. OLED만 완성한 팀도 충분히 성취감을 느끼도록 "OLED 대시보드를 완성한 것만으로도 엣지 컴퓨팅을 구현한 것"이라고 격려하세요. PC 연동까지 성공한 팀은 학급 전체에 시연하도록 하면 자연스러운 피어 러닝이 됩니다.</p>
</div>

## 차별화 전략

<div class="differentiation">
  <div class="fast-learner">
    <p><strong>빠른 학생에게:</strong></p>
    <ul>
      <li>deque 버퍼 크기를 30, 60, 120으로 바꿔보며 그래프 범위 차이를 실험하세요</li>
      <li>serial 통신 데이터 형식을 표준화하여 다른 팀도 같은 PC 대시보드 코드를 쓸 수 있도록 제안해 보세요</li>
      <li>OLED에 미니 그래프(최근 20개 데이터 꺾은선)를 추가해 보세요</li>
      <li>다른 팀의 serial 통신 오류를 진단하고 도와주세요</li>
    </ul>
  </div>
  <div class="slow-learner">
