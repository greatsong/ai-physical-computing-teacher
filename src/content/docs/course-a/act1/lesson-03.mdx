<details class="student-content">
  <summary>학생 교재 — CSV 자동 저장 코드</summary>
  <div class="content">

```python
from machine import I2C, Pin
import time

# I2C 통신 설정 (Grove Shield I2C0 포트)
i2c = I2C(0, sda=Pin(8), scl=Pin(9))

# DHT20 센서 초기화
sensor_addr = 0x38
time.sleep(0.1)

def read_dht20():
    """DHT20 센서에서 온도와 습도를 읽습니다"""
    i2c.writeto(sensor_addr, bytes([0xAC, 0x33, 0x00]))
    time.sleep(0.08)
    data = i2c.readfrom(sensor_addr, 7)

    humidity = ((data[1] << 12) | (data[2] << 4) | (data[3] >> 4)) / 1048576 * 100
    temperature = (((data[3] & 0x0F) << 16) | (data[4] << 8) | data[5]) / 1048576 * 200 - 50

    return round(temperature, 1), round(humidity, 1)

# CSV 파일 생성 (헤더 쓰기)
file = open("data.csv", "w")
file.write("측정번호,온도(C),습도(%)\n")

# 10초마다 측정하여 파일에 저장
count = 1
while True:
    temp, humi = read_dht20()

    # 화면에 출력
    print(f"[{count}] 온도: {temp}°C, 습도: {humi}%")

    # 파일에 저장
    file.write(f"{count},{temp},{humi}\n")
    file.flush()  # 버퍼를 비워 즉시 저장

    count += 1
    time.sleep(10)
```

  <p>"자, 이제 모둠에서 이야기해봅시다. 사람이 직접 기록하는 것과 코드가 자동으로 기록하는 것, 뭐가 다를까요?"</p>
  <p>"힌트를 드릴게요. 정확성, 간격의 일정성, 빠짐없는 기록, 24시간 가동 — 이런 관점에서 생각해보세요."</p>
  <p>"그리고 자동 기록에도 한계가 있을까요? 센서가 고장 나면? 저장 공간이 부족하면?"</p>
</div>

<div class="student-voice">
  <span class="bubble">"사람은 깜빡할 수 있는데 코드는 안 빠트려요"</span>
  <span class="bubble">"10초마다 정확하게 기록되잖아요"</span>
  <span class="bubble">"근데 센서가 고장 나면 잘못된 데이터를 계속 기록하겠죠?"</span>
  <span class="bubble">"농장에서 온도 관리할 때 쓸 수 있을 것 같아요"</span>
</div>

<div class="teaching-tip">
  <p>토론에서 "표본 간격의 일정성"과 "인간 오차 제거"라는 키워드가 나오도록 유도하세요. 모든 팀이 발표할 필요는 없고, 핵심적인 의견 2&#126;3개를 골라 전체에 공유하면 됩니다.</p>

## 정리 (10분)

<div class="teacher-script">
  <p>"오늘 수업을 정리해봅시다. 1차시에서 온도를 읽고, 2차시에서 빛을 그래프로 보고, 오늘 3차시에서 데이터를 파일로 저장했습니다."</p>
  <p>"오늘 배운 핵심은 <strong>세 단계</strong>예요. 파일을 열고(<code>open</code>), 데이터를 쓰고(<code>write</code>), 즉시 저장한다(<code>flush</code>). 이 세 단계만 기억하면 됩니다."</p>
  <p>"여러분이 만든 30줄짜리 데이터 — 이건 그냥 숫자가 아니에요. 여러분이 손을 올렸을 때 온도가 올라가고, 입김을 불었을 때 습도가 올라간 <strong>이야기</strong>가 담겨 있습니다. 데이터를 읽는다는 건 그 이야기를 찾아가는 과정입니다."</p>
  <p>"기상청의 자동 기상 관측소, 병원의 환자 모니터링, 공장의 품질 관리 — 모두 오늘 여러분이 만든 것과 같은 원리입니다. 사람이 24시간 지켜볼 수 없는 것을 코드가 대신 기록합니다."</p>
  <p>"다음 시간에는 드디어 <strong>OLED 디스플레이</strong>를 사용합니다. Shell 창이 아니라 작은 화면에 센서 데이터를 보여주는 나만의 <strong>대시보드</strong>를 만들어볼 거예요. 1막의 마지막 차시입니다!"</p>
</div>

<div class="teaching-tip">
  <p>1&#126;3차시의 흐름을 한 줄로 정리해주세요. "센서로 읽고(1차시) → 그래프로 보고(2차시) → 파일로 기록한다(3차시)." 이 흐름이 4차시의 대시보드까지 이어지면서, 1막 전체가 하나의 스토리로 연결된다는 것을 인식시키세요.</p>
</div>

## 차별화 전략

<div class="differentiation">
  <div class="fast-learner">
    <p><strong>빠른 학생에게:</strong></p>
    <ul>
      <li>도전 코드 실행: <code>for</code> 반복문으로 30회 자동 종료 + 파일 크기 확인</li>
      <li>CSV에 타임스탬프를 추가하는 코드를 자발적으로 작성해보기</li>
      <li>30개 데이터의 최대/최소/평균을 계산해보기</li>
      <li>"이 자동 기록 시스템을 과학 시간 실험에 활용하면 어떨까?" 생각해보기</li>
    </ul>
  </div>
  <div class="slow-learner">
    <p><strong>느린 학생에게:</strong></p>
    <ul>
      <li>코드를 복사/붙여넣기해서 실행하는 것부터 시작</li>
      <li>CSV 파일이 만들어졌다는 것만 확인하면 성공</li>
      <li>"파일을 열고, 쓰고, 저장한다" 세 단계만 기억하게 하기</li>
      <li>파일 내용을 함께 읽으며 "이 숫자가 온도고, 이 숫자가 습도야" 확인</li>
    </ul>
  </div>
</div>
