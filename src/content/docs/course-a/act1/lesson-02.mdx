<div class="teacher-script">
  <p>"이제 코드를 입력합니다. 스크린을 보세요."</p>
  <p>"이번 코드에서 핵심은 <code>read_u16()</code>이에요. u16이 뭘까요? unsigned 16-bit, 즉 부호 없는 16비트 정수라는 뜻입니다. 0부터 65535까지의 값이 나와요."</p>
  <p>"65535가 왜 이 숫자냐면, 2의 16승이 65536이거든요. 0부터 세니까 최대가 65535입니다. 수학 시간에 거듭제곱 배웠죠? 바로 그겁니다."</p>
  <p>"코드를 입력하고 실행 버튼 누르세요."</p>
</div>

<details class="student-content">
  <summary>학생 교재 — 기본 코드</summary>
  <div class="content">

```python
from machine import ADC, Pin
import time

# 아날로그 입력 설정 (Grove Shield A0 포트 = GP26)
light = ADC(Pin(26))

# 2초마다 밝기 측정하여 출력
while True:
    raw_value = light.read_u16()           # 0 &#126; 65535
    brightness = round(raw_value / 65535 * 100, 1)  # 퍼센트로 변환
    print(f"밝기: {brightness}%")
    print(brightness)  # Plotter용 숫자만 출력
    time.sleep(2)
```

  </div>
</details>

<div class="teaching-tip">
  <p><strong>코드 해설 포인트</strong> — 실행 후 3가지만 짚어주세요:</p>
  <ol>
    <li><code>ADC(Pin(26))</code> — "ADC는 아날로그를 디지털로 바꿔주는 변환기예요. 26번 핀이 A0 포트에 연결되어 있어요."</li>
    <li><code>read_u16()</code> — "0&#126;65535 사이의 숫자로 읽어요. 어두우면 0에 가깝고, 밝으면 65535에 가까워요."</li>

## 정리 (10분)

<div class="teacher-script">
  <p>"오늘 수업을 정리해봅시다. 지난 시간에는 온도를 숫자로, 오늘은 빛을 숫자로 바꿨습니다."</p>
  <p>"지난 시간과 다른 점이 뭐였죠? 맞아요, 포트가 달랐어요. 온도 센서는 I2C 포트, 빛 센서는 A0 포트. 센서마다 통신 방식이 다르기 때문입니다."</p>
  <p>"그리고 오늘은 <strong>그래프</strong>를 처음 그려봤습니다. 숫자만 보면 '47%'가 높은 건지 낮은 건지 감이 잘 안 오는데, 그래프로 보면 변화가 한눈에 보이죠? 이게 바로 <strong>시각화</strong>의 힘입니다."</p>
  <p>"여러분 기록표에서 체감 밝기와 센서 값이 다른 곳이 있었나요? 우리 눈은 자동으로 밝기에 적응하지만, 센서는 그렇지 않아요. 센서가 주는 객관적인 숫자 — 이것이 데이터의 가치입니다."</p>
  <p>"다음 시간에는 센서 데이터를 <strong>파일로 자동 저장</strong>합니다. 화면에 흘러가는 데이터를 영구적으로 기록하는 '자동 기록 시스템'을 만들어볼 거예요."</p>
</div>

<div class="teaching-tip">
  <p>1차시(온도)와 2차시(빛)의 공통점과 차이점을 정리해주세요. 공통점: "느낌을 숫자로 바꾸고, 데이터로 비교한다." 차이점: "센서 종류, 포트, 통신 방식이 다르다." 이 패턴을 인식하면 3차시 이후 새로운 센서를 만나도 학생들이 스스로 접근할 수 있습니다.</p>
</div>

## 차별화 전략

<div class="differentiation">
  <div class="fast-learner">
    <p><strong>빠른 학생에게:</strong></p>
    <ul>
      <li>도전 코드 실행: 밝기 판정(밝음/보통/어두움) + Pico 내장 LED 연동</li>
      <li>"65535가 왜 이 숫자인지 계산해보세요" — 2의 16승 = 65536, 0부터 세면 최대 65535</li>
      <li>교실 조도 기준 조사: 학교 교실에 적정한 조도(lux)가 얼마인지 찾아보기</li>
      <li>Plotter 그래프의 기울기와 수학 시간 '변화율' 개념의 연결</li>
    </ul>
  </div>
  <div class="slow-learner">
    <p><strong>느린 학생에게:</strong></p>
    <ul>
      <li>코드를 복사/붙여넣기해서 실행하는 것부터 시작</li>
      <li>센서를 손으로 가리고 떼면서 Plotter 그래프 변화만 관찰</li>
      <li>"올라가면 밝아지는 거, 내려가면 어두워지는 거"만 이해해도 충분</li>
      <li>측정은 3곳만 해도 괜찮음</li>
    </ul>
  </div>
</div>
