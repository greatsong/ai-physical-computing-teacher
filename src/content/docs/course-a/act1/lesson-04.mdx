<details class="student-content">
  <summary>학생 교재 — 대시보드 기본 코드</summary>
  <div class="content">

```python
from machine import I2C, Pin, ADC
from ssd1306 import SSD1306_I2C
import time

# I2C1 설정
i2c = I2C(1, sda=Pin(6), scl=Pin(7))

# OLED 디스플레이 초기화 (128x64)
oled = SSD1306_I2C(128, 64, i2c)

# DHT20 센서
sensor_addr = 0x38
time.sleep(0.1)

# 빛 센서 (A0)
light = ADC(Pin(26))

def read_dht20():
    i2c.writeto(sensor_addr, bytes([0xAC, 0x33, 0x00]))
    time.sleep(0.08)
    data = i2c.readfrom(sensor_addr, 7)
    humidity = ((data[1] << 12) | (data[2] << 4) | (data[3] >> 4)) / 1048576 * 100
    temperature = (((data[3] & 0x0F) << 16) | (data[4] << 8) | data[5]) / 1048576 * 200 - 50
    return round(temperature, 1), round(humidity, 1)

# 대시보드 루프
while True:
    temp, humi = read_dht20()
    bright = round(light.read_u16() / 65535 * 100, 1)

    oled.fill(0)  # 화면 지우기
    oled.text("=== Dashboard ===", 0, 0)
    oled.text(f"Temp:  {temp} C", 0, 16)
    oled.text(f"Humi:  {humi} %", 0, 28)
    oled.text(f"Light: {bright} %", 0, 40)
    oled.show()  # 화면 전송

    time.sleep(1)
```

<div class="patrol-point">
  <p><strong>순회 시 확인:</strong></p>
  <ol>
    <li>기본 대시보드를 벗어나 레이아웃을 수정하거나 요소를 추가하고 있는가</li>
    <li>좌표(x, y) 값을 바꿔보며 배치를 실험하고 있는가</li>

<div class="teaching-tip">
  <p>대시보드 디자인은 정답이 없습니다. "같은 데이터라도 배치에 따라 읽는 속도가 달라진다"는 점을 강조하세요. "만약 이 대시보드를 교실 벽에 걸어둔다면, 멀리서도 온도를 바로 확인할 수 있으려면 어떻게 배치해야 할까요?"와 같은 질문으로 디자인 사고를 유도하세요.</p>

## 정리 (10분)

<div class="teacher-script">
  <p>"여러분, 1막이 끝났습니다. 4차시에 걸쳐 무엇을 했는지 한번 돌아봅시다."</p>
  <p>"1차시 — 센서로 온도를 읽었습니다. 느낌을 숫자로 바꿨죠."</p>
  <p>"2차시 — 빛을 숫자로 바꾸고, 그래프로 시각화했습니다."</p>
  <p>"3차시 — 데이터를 파일에 자동 저장했습니다. 코드가 대신 기록하는 시스템이었죠."</p>
  <p>"4차시 — 오늘, 세 가지를 하나로 합쳐서 대시보드를 만들었습니다."</p>
  <p>"여러분 앞에 있는 이 작은 OLED 화면 — 이게 바로 자동차 계기판, 병원 환자 모니터, 공장 모니터링 시스템의 원리입니다. 128x64 픽셀 안에 여러분이 그 원리를 직접 구현한 겁니다."</p>
  <p>"그런데 한 가지 느껴본 것 있지 않나요? 코드를 한 줄 한 줄 직접 쓰는 건... 좀 힘들죠?"</p>
</div>

<div class="student-voice">
  <span class="bubble">"맞아요, 오타 때문에 에러 많이 났어요"</span>
  <span class="bubble">"코드가 점점 길어져서 헷갈려요"</span>
</div>

<div class="teacher-script">
  <p>"맞아요. 그래서 다음 시간부터는 방법이 바뀝니다. <strong>2막: 바이브 코딩</strong>이 시작됩니다!"</p>
  <p>"지금까지 직접 쓴 코드를 AI에게 자연어로 설명하면 어떤 코드가 나올까요? AI가 생성한 코드와 여러분의 코드를 비교해봅니다. 기대하세요!"</p>
</div>

<div class="teaching-tip">
  <p>1막 마무리는 <strong>성취감 + 다음 막에 대한 기대</strong>의 균형이 중요합니다. "4차시 동안 센서 연결, 코드 작성, 데이터 기록, 대시보드 설계까지 직접 했다"는 성취감을 충분히 느끼게 한 뒤, "이걸 AI에게 시키면?"이라는 호기심으로 2막을 예고하세요. 1막에서 직접 코딩한 경험이 있어야 2막에서 AI 코드의 의미를 이해할 수 있습니다.</p>
</div>

## 차별화 전략

<div class="differentiation">
  <div class="fast-learner">
    <p><strong>빠른 학생에게:</strong></p>
    <ul>
      <li>도전 코드 실행: 구분선 + 쾌적도 판정 + 온도 미니 바 그래프</li>
      <li>128x64 OLED 레이아웃에 미술 시간의 시각적 위계(크기, 위치, 대비) 적용</li>
      <li>막대 그래프 높이 비례식을 정확히 계산해보기: <code>(t - 15) / 20 * 12</code>의 의미</li>
      <li>병원 모니터와 자동차 계기판의 공통 설계 원리 분석</li>
    </ul>
  </div>
  <div class="slow-learner">
    <p><strong>느린 학생에게:</strong></p>
    <ul>
      <li>기본 코드를 복사/붙여넣기해서 실행하는 것부터 시작</li>
      <li>OLED에 세 가지 값이 나오면 성공 — 레이아웃 꾸미기는 선택</li>
      <li>센서를 만지며 화면 값이 변하는 것을 관찰하는 데 집중</li>
      <li>"1차시 센서 + 2차시 센서 + 화면 = 대시보드"라는 통합 개념만 이해해도 충분</li>
    </ul>
  </div>
</div>
