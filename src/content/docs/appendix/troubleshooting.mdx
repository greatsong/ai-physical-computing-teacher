## 하드웨어 문제

| 증상 | 원인 | 대응 |
|------|------|------|
| Pico 인식 안 됨 | USB 케이블 (충전 전용) | 데이터 케이블로 교체 |
| 센서 값이 0 고정 | 포트 오류 (A0에 I2C 센서) | I2C 포트로 이동 |
| `OSError: [Errno 5]` | I2C 접촉 불량 | 케이블 재연결 |
| OLED 화면 안 나옴 | I2C 주소 충돌 또는 전원 | I2C 스캔 코드로 주소 확인 |
| 부저에서 소리 안 남 | PWM 주파수 0 또는 핀 오류 | 핀 번호와 `duty_u16` 값 확인 |

## I2C 관련 문제

| 증상 | 원인 | 대응 |
|------|------|------|
| `OSError: [Errno 5] EIO` | I2C 접촉 불량 또는 잘못된 포트 | 케이블 재연결, I2C 스캔 실행 |
| `i2c.scan()` 결과가 `[]` | 센서 미연결 또는 전원 문제 | 케이블 확인, 포트 변경, 센서 전원 확인 |
| `i2c.scan()` 주소 다름 | 잘못된 I2C 버스 또는 핀 | `I2C(1, sda=Pin(6), scl=Pin(7))` 확인 |
| OLED에 아무것도 안 나옴 | `oled.show()` 누락 또는 좌표 오류 | `oled.show()` 호출 확인 |
| 센서 2개 동시 사용 시 오류 | 같은 I2C 주소 충돌 | `i2c.scan()`으로 주소 확인 (예: TSL2591/TCS34725/VL53L0X 모두 0x29) |

### I2C 진단 코드

학생이 I2C 관련 문제를 호소하면, 이 코드부터 실행하게 하세요:

```python
# I2C 연결 진단 코드 — 학생에게 이 코드부터 실행하게 하세요
from machine import I2C, Pin
i2c = I2C(1, sda=Pin(6), scl=Pin(7))
devices = i2c.scan()
if devices:
    print("발견된 장치:", [hex(d) for d in devices])
else:
    print("장치 없음 — 케이블과 포트를 확인하세요")
```

## WiFi 관련 문제 (코스 B)

| 증상 | 원인 | 대응 |
|------|------|------|
| WiFi 연결 실패 | 5GHz 네트워크 | Pico는 2.4GHz만 지원. 공유기 설정 확인 |
| WiFi 연결 실패 | 학교 네트워크 인증 | 스마트폰 핫스팟 사용 |
| WiFi 연결 매우 느림 | 다수 학생 동시 연결 | 핫스팟 분산 사용 (4~5개 그룹) |
| Google Sheets 전송 실패 | 302 리다이렉트 미처리 | 리다이렉트 처리 코드 사용 (교재 참고) |
| Google Sheets 전송 실패 | Apps Script 독립 실행형 | Sheets > Extensions > Apps Script로 재생성 |
| Flask 서버 연결 실패 | PC와 Pico가 다른 네트워크 | 같은 WiFi에 연결 확인, IP 주소 확인 |

## 에러 메시지 빠른 해석표

| 에러 메시지 | 의미 | 교사 대응 |
|------------|------|----------|
| `ENOMEM` | 메모리 부족 | Pico 리셋 (Ctrl+D), 코드 축소 |
| `ETIMEDOUT` | 연결 시간 초과 | 케이블/WiFi 확인 |
| `bad SCL pin` | 잘못된 I2C 핀 | `I2C(1, sda=Pin(6), scl=Pin(7))` 확인 |
| `module not found` | 라이브러리 없음 | Thonny에서 ssd1306 패키지 설치 |
| `ECONNREFUSED` | 서버 거부 | Flask 서버가 실행 중인지 확인 |
| `invalid syntax` | 문법 오류 | 콜론, 괄호, 들여쓰기 확인 |
