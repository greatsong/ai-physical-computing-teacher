## 하드웨어 문제

| 증상 | 원인 | 대응 |
|------|------|------|
| Pico 인식 안 됨 | USB 케이블 (충전 전용) | 데이터 케이블로 교체 |
| 센서 값이 0 고정 | 포트 오류 (A0에 I2C 센서) | I2C 포트로 이동 |
| `OSError: [Errno 5]` | I2C 접촉 불량 | 케이블 재연결 |
| OLED 화면 안 나옴 | I2C 주소 충돌 또는 전원 | I2C 스캔 코드로 주소 확인 |
| 부저에서 소리 안 남 | PWM 주파수 0 또는 핀 오류 | 핀 번호와 `duty_u16` 값 확인 |

## 소프트웨어 문제

| 증상 | 원인 | 대응 |
|------|------|------|
| `SyntaxError` | 들여쓰기/콜론/따옴표 | 에러 메시지의 줄 번호 확인 |
| `ImportError` | 라이브러리 미설치 | Thonny 패키지 관리자에서 설치 |
| `MemoryError` | Pico 메모리 부족 | 코드 축소 또는 Pico 리셋 |
| Plotter 안 나옴 | print 형식 오류 | `print(값)` 형태만 Plotter 지원 |

## 바이브 코딩 문제 (2막~)

| 증상 | 원인 | 대응 |
|------|------|------|
| AI 코드가 안 돌아감 | 센서 주소/핀 번호 오류 | I2C 스캔으로 실제 주소 확인 |
| AI가 없는 라이브러리 사용 | AI가 표준 Python 기준으로 생성 | MicroPython 호환 라이브러리로 수정 |
| AI 코드가 너무 복잡 | 프롬프트가 모호 | 요구사항을 구체적으로 재작성 |
