## 교사용 참고사항

### HTML 크기 주의

<Aside type="caution">
- 시각화 코드가 추가되어 HTML이 커집니다
- `gc.collect()` 확인, 필요시 `send()` 분할
- 학생이 바이브 코딩으로 많이 추가하면 메모리 이슈 가능 — CSS 간소화 안내
</Aside>

### Canvas vs SVG

| 방식 | 장점 | 적합한 상황 |
|------|------|-------------|
| Canvas | 메모리 효율적, 복잡한 그래프에 유리 | 실시간 데이터 차트, 게임 |
| SVG | 확대해도 선명, DOM으로 접근 가능 | 간단한 차트, 인터랙션 |

<div class="patrol-point">
  <ol>
    <li>기본 코드는 Canvas를 사용합니다 (메모리 효율적)</li>
    <li>AI가 SVG를 제안할 수도 있음 — 둘 다 유효하니 학생 선호에 맡기세요</li>
    <li>바이브 코딩 후 페이지가 너무 커지면 CSS 최소화 또는 기능 분리를 안내하세요</li>
  </ol>
</div>

### 흔한 문제 & 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 그래프 빈 화면 | 데이터 미축적 | 30초 이상 대기 후 확인 |
| `ENOMEM` 에러 | HTML 크기 초과 | `gc.collect()`, CSS 간소화, send() 분할 |
| 게이지 바늘 안 움직임 | JS 함수 오류 | 에러 메시지를 AI에게 보내서 해결 |
| 채널 차트 빈 칸 | AP 스캔 지연 | AP 스캔 완료 후 자동 갱신 대기 |
