### 파일 1: wifi_config.py

<div class="code-file-label">wifi_config.py — Pico에 저장</div>

```python
# wifi_config.py
# Pico에 저장해두면 다른 코드에서 import해서 쓸 수 있어요
WIFI_SSID = "교실Wi-Fi이름"
WIFI_PASSWORD = "비밀번호"
```

<Tabs>
  <TabItem label="🏫 학교에서">
    학교 Wi-Fi는 **비밀번호가 설정**되어 있어요. 선생님이 SSID와 비밀번호를 칠판에 적어줄 거예요. Pico는 **2.4GHz만 지원**하므로 5GHz 전용 네트워크는 사용할 수 없어요.

    ```python
    # wifi_config.py — 학교 버전
    WIFI_SSID = "학교Wi-Fi이름"        # 선생님이 알려준 SSID
    WIFI_PASSWORD = "학교Wi-Fi비밀번호"  # 선생님이 알려준 비밀번호
    ```

    <Aside type="tip" title="IP를 고정하고 싶다면">
    DHCP가 자동으로 IP를 바꾸기 때문에, 매번 IP가 달라질 수 있어요. 아래처럼 **정적 IP**를 설정하면 항상 같은 주소로 접속할 수 있습니다.

    ```python
    # wifi_config.py — 학교 정적 IP 버전
    WIFI_SSID = "학교Wi-Fi이름"
    WIFI_PASSWORD = "학교Wi-Fi비밀번호"

    STATIC_IP = "172.30.6.10"    # 다른 기기와 안 겹치는 번호
    SUBNET    = "255.255.0.0"    # 학교 네트워크는 보통 /16 대역
    GATEWAY   = "172.30.0.1"     # 기본 게이트웨이
    DNS       = "8.8.8.8"
    ```

    `main.py`에서 Wi-Fi 연결 직후에 한 줄 추가:

    ```python
    wlan.connect(WIFI_SSID, WIFI_PASSWORD)
    while not wlan.isconnected():
        time.sleep(1)
    # ↓ 이 줄을 추가하면 IP 고정!
    wlan.ifconfig((STATIC_IP, SUBNET, GATEWAY, DNS))
    ```
    </Aside>

    **학교 IP 대역 확인법:** Thonny 셸에서 연결 후 `wlan.ifconfig()`를 실행하면 자동 할당된 IP, 서브넷, 게이트웨이를 확인할 수 있어요. 정적 IP를 설정할 때 이 값들을 참고하세요.
  </TabItem>
  <TabItem label="🏠 집에서">
    집에 가져가서 실습하려면 `wifi_config.py`만 바꾸면 돼요! 집 공유기의 **2.4GHz SSID**를 확인하세요 (5GHz와 이름이 다를 수 있어요).

    **1단계: 기본 연결 (이것만 하면 바로 동작!)**

    ```python
    # wifi_config.py — 집 버전
    WIFI_SSID = "우리집Wi-Fi이름"
    WIFI_PASSWORD = "우리집비밀번호"
    ```

    이렇게만 바꾸고 실행하면 Pico가 자동으로 IP를 받아요. Thonny 셸에 뜨는 IP로 접속하면 끝!

    **2단계 (선택): 내 네트워크 정보 확인하기**

    </Aside>

    공유기마다 대역이 다르므로 **반드시 직접 확인**하세요:

    | 공유기 | 흔한 대역 | 서브넷 |
    |--------|----------|--------|
    | iptime | `192.168.0.x` | `255.255.255.0` |
    | KT/SKT/LG | `172.30.x.x` 또는 `192.168.x.x` | 다양함 |
    | TP-Link, ASUS | `192.168.0.x` 또는 `192.168.1.x` | `255.255.255.0` |

    Pico 없이 PC/스마트폰에서 확인하는 방법:
    - **Windows:** `Win+R` → `cmd` → `ipconfig` 입력 → **IPv4 주소**, **서브넷 마스크**, **기본 게이트웨이** 확인
    - **Mac:** Option(⌥) 키를 누른 채 메뉴바 Wi-Fi 아이콘 클릭 → **IP 주소**, **서브넷 마스크**, **라우터** 표시됨
    - **iPhone:** 설정 → Wi-Fi → 연결된 네트워크 옆 ⓘ 버튼 → **IP 주소**, **서브넷 마스크**, **라우터**
    - **Android:** 설정 → Wi-Fi → 연결된 네트워크 길게 누르기 → 네트워크 세부정보

    **3단계 (선택): 정적 IP 설정**

    2단계에서 확인한 값을 기반으로 설정합니다:

    ```python
    # wifi_config.py — 집 정적 IP 버전
    # (아래 숫자들은 예시! 반드시 2단계에서 확인한 값으로 바꾸세요)
    WIFI_SSID = "우리집Wi-Fi이름"
    WIFI_PASSWORD = "우리집비밀번호"

    STATIC_IP = "192.168.0.200"    # 내 IP의 마지막 숫자만 200~250으로 변경
    SUBNET    = "255.255.255.0"    # 서브넷은 그대로 복사
    GATEWAY   = "192.168.0.1"      # 게이트웨이도 그대로 복사
    DNS       = "8.8.8.8"          # 구글 DNS (고정값)
    ```

    <Aside type="tip" title="STATIC_IP는 어떻게 정하나요?">
    `wlan.ifconfig()`로 확인한 **내 IP의 마지막 숫자만 바꾸면** 돼요. 예를 들어 자동 할당이 `192.168.0.15`였다면, 정적 IP는 `192.168.0.200`처럼 **200~250 사이** 높은 번호를 쓰세요. 이 범위는 보통 다른 기기가 쓰지 않아요. 나머지 값(서브넷, 게이트웨이)은 확인한 그대로 복사합니다.
    </Aside>

    `main.py`는 수정할 필요 없어요! `wifi_config.py`만 교체하면 어디서든 동일하게 작동합니다.
  </TabItem>

# ============================================
# 교실 Wi-Fi 모니터 - 1차시 기본 코드
# Pico 2 WH 웹서버 + 자기소개 페이지
# ============================================

import network
import socket
import time
from wifi_config import WIFI_SSID, WIFI_PASSWORD

# ── Wi-Fi 연결 ──────────────────────────────
wlan = network.WLAN(network.STA_IF)
wlan.active(True)

print("Wi-Fi 연결 중...")
wlan.connect(WIFI_SSID, WIFI_PASSWORD)

wait = 0
while not wlan.isconnected():
    print(f"  대기 중... ({wait}초)")
    time.sleep(1)
    wait += 1
    if wait > 15:
        print("연결 실패! wifi_config.py를 확인하세요.")
        raise SystemExit

my_ip = wlan.ifconfig()[0]
rssi = wlan.status('rssi')
print(f"연결 완료! IP: {my_ip} / RSSI: {rssi} dBm")

# ── HTML 페이지 ─────────────────────────────
def make_page():
    rssi = wlan.status('rssi')
    uptime = time.ticks_ms() // 1000

    return f"""<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Pico 모니터</title>

        body {{

        }}
        .header {{
            text-align: center;
            padding: 20px 0;
        }}
        .header h1 {{
            color: #00d4ff;
            font-size: 1.6em;
            margin-bottom: 5px;
        }}
        .header p {{
            color: #666;
            font-size: 0.9em;
        }}
        .grid {{
            display: grid;
            grid-template-columns: 1fr 1fr;
            gap: 12px;
            max-width: 500px;
            margin: 20px auto;
        }}
        .card {{
            background: #1a1a3e;
            border-radius: 12px;
            padding: 18px;
            text-align: center;
            border: 1px solid #2a2a5e;
        }}
        .card.wide {{
            grid-column: span 2;
        }}
        .card .icon {{

            font-weight: bold;
        }}
        .footer {{
            text-align: center;
            margin-top: 30px;
            color: #444;
            font-size: 0.8em;
        }}
    </style>
</head>
<body>
    <div class="header">
        <h1>Pico Wi-Fi 모니터</h1>
        <p>Raspberry Pi Pico 2 WH</p>
    </div>

    <div class="grid">
        <div class="card">
            <div class="icon">📶</div>
            <div class="label">Wi-Fi 신호</div>
            <div class="value">{rssi} dBm</div>
        </div>

        <div class="card">
            <div class="icon">🌐</div>
            <div class="label">IP 주소</div>
            <div class="value" style="font-size:1em;">{my_ip}</div>
        </div>

        <div class="card">
            <div class="icon">⏱️</div>
            <div class="label">가동 시간</div>
            <div class="value">{uptime}초</div>
        </div>

        <div class="card">
            <div class="icon">👤</div>
            <div class="label">만든 사람</div>
            <div class="value">이름</div>

        </div>
    </div>

    <div class="footer">
        Powered by MicroPython on Pico 2 WH
    </div>
</body>
</html>"""

# ── 웹서버 시작 ─────────────────────────────
server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
server.bind(('0.0.0.0', 80))
server.listen(2)

print()
print("=" * 45)
print(f"  웹서버 시작! http://{my_ip}")
print("  브라우저에서 접속하세요")
print("=" * 45)

while True:
    try:
        client, addr = server.accept()
        request = client.recv(1024)

        # favicon 요청 무시
        req_str = request.decode()
        if 'favicon' in req_str:
            client.close()
            continue

        print(f"접속: {addr[0]}")
        page = make_page()

        client.send(b"HTTP/1.1 200 OK\r\nContent-Type: text/html; charset=utf-8\r\nConnection: close\r\n\r\n")
        client.send(page.encode())
        client.close()

    except Exception as e:
        print(f"에러: {e}")
        try:
            client.close()
        except:
            pass

### 네트워크 개념 — 교사 배경지식

학생 교재에서는 IP 주소를 **색상 구분 다이어그램**으로 시각화합니다. 학생이 질문하면 아래 내용으로 보충 설명해주세요.

<div class="compare-cards">
  <div class="compare-card old" style="flex:1;">
    <div class="compare-title">서브넷 255.255.255.0 (가정)</div>
    <div class="compare-item">앞 3자리가 "동네 주소"</div>
    <div class="compare-item">마지막 1자리가 "기기 번호"</div>
    <div class="compare-item">같은 동네 = 최대 254대</div>
  </div>
  <div class="compare-arrow">vs</div>
  <div class="compare-card new" style="flex:1;">
    <div class="compare-title">서브넷 255.255.0.0 (학교)</div>
    <div class="compare-item">앞 2자리가 "동네 주소"</div>
    <div class="compare-item">뒤 2자리가 "기기 번호"</div>
    <div class="compare-item">같은 동네 = 최대 65,534대</div>
  </div>
</div>

<div class="key-terms">
  <dl>
    <dt>IP 주소 (Internet Protocol Address)</dt>
    <dd>네트워크에서 각 기기를 식별하는 고유 주소. <code>192.168.0.15</code>처럼 4개 숫자(0~255)를 점으로 구분. 같은 네트워크에 같은 IP가 두 개 있으면 <strong>IP 충돌</strong>이 발생하여 둘 다 접속 불가.</dd>
    <dt>서브넷 마스크 (Subnet Mask)</dt>
    <dd>"같은 네트워크"의 범위를 결정하는 규칙. 255 = "이 자리는 네트워크 주소(고정)", 0 = "이 자리는 기기 번호(변동)". 학생에게는 <strong>"앞 3자리가 같으면 같은 네트워크"</strong>라고 안내하면 충분합니다.</dd>
    <dt>게이트웨이 (Gateway)</dt>
    <dd>내부 네트워크와 외부 인터넷을 연결하는 출입구 = <strong>공유기(라우터)의 IP 주소</strong>. 보통 서브넷의 첫 번째 주소(예: <code>.1</code>). 브라우저에 게이트웨이 IP를 입력하면 공유기 관리 페이지가 뜸.</dd>
    <dt>DHCP (Dynamic Host Configuration Protocol)</dt>
    <dd>공유기가 연결된 기기에 <strong>자동으로 IP를 배정</strong>하는 기능. 기본 코드에서는 DHCP로 자동 IP를 받음. 정적 IP를 설정하면 DHCP를 무시하고 지정한 IP를 사용.</dd>
    <dt>DNS (Domain Name System)</dt>
    <dd><code>google.com</code> 같은 도메인 이름을 IP 주소로 변환하는 서비스. <code>8.8.8.8</code>은 구글이 운영하는 공개 DNS. Pico 프로젝트에서는 직접 사용하지 않지만, <code>wlan.ifconfig()</code>에 포함됨.</dd>
  </dl>
</div>

<Aside type="tip" title="학생 질문 예상 답변">

**"서브넷 255가 뭔가요?"** → "255는 '이 자리는 바뀌면 안 되는 네트워크 주소'라는 뜻이에요. 0은 '이 자리는 기기마다 다른 번호'라는 뜻이고요."

**"학교는 왜 255.255.0.0인가요?"** → "학교는 기기가 훨씬 많으니까 뒤의 2자리를 기기 주소로 쓰는 거예요. 가정은 254대, 학교는 65,534대까지 가능해요."

**"왜 같은 Wi-Fi에 연결해야 해요?"** → "Pico 웹서버는 인터넷에 공개된 게 아니라 같은 공유기 안에서만 접속할 수 있어요. 학생 교재 다이어그램처럼, 같은 공유기에 연결된 기기들끼리만 서로 찾을 수 있어요."

**"게이트웨이를 왜 써야 해요?"** → "Pico끼리 통신할 때는 안 써도 되지만, 정적 IP를 설정할 때 빈칸으로 두면 에러가 나서 넣어주는 거예요."
</Aside>

### Wi-Fi 환경별 설정 가이드

<Aside type="caution" title="Pico 2 WH는 2.4GHz 전용!">
Pico 2 WH의 CYW43439 칩은 **2.4GHz Wi-Fi만 지원**합니다. 5GHz 전용 네트워크에는 연결할 수 없어요. 학교·집 모두 **2.4GHz 대역의 SSID**를 확인하세요. 최근 공유기는 2.4G와 5G를 같은 이름으로 통합하는 경우도 있는데, 이때 Pico는 자동으로 2.4G에 연결됩니다.
</Aside>

#### 학교 네트워크

| 항목 | 값 | 비고 |
|------|-----|------|
| SSID | 학교 관리자에게 확인 | **2.4GHz 대역** SSID 사용 |
| 비밀번호 | 있음 | 수업 전 칠판에 게시 |
| IP 대역 | `172.30.x.x` | 넓은 /16 대역 |
| 서브넷 마스크 | `255.255.0.0` | |
| 게이트웨이 | `172.30.0.1` (교실마다 다를 수 있음) | |

**정적 IP 선택 시 주의:**
- DHCP 자동 배포 범위(보통 100~200)를 피해 **낮은 번호(10~30)** 또는 **높은 번호(240~250)** 선택
- 같은 교실에서 여러 Pico를 쓸 경우 학생마다 다른 번호 배정 (예: `172.30.6.10`, `172.30.6.11`, ...)
- AP마다 VLAN이 다를 수 있으므로, 연결 후 `wlan.ifconfig()`로 자동 할당된 게이트웨이를 먼저 확인

#### 가정 네트워크

공유기 제조사마다 IP 대역이 다릅니다. **특정 대역을 단정 짓지 말고**, 학생에게 `wlan.ifconfig()`로 직접 확인하도록 안내하세요.

| 공유기 | 흔한 대역 | 서브넷 | 게이트웨이 |
|--------|----------|--------|-----------|
| iptime | `192.168.0.x` | `255.255.255.0` | `192.168.0.1` |
| KT 기가인터넷 | `172.30.x.x` | `255.255.0.0` | `172.30.0.1` |
| SKT/LG | `192.168.x.x` 또는 `172.x.x.x` | 다양 | 다양 |
| TP-Link, ASUS | `192.168.0.x` 또는 `192.168.1.x` | `255.255.255.0` | `x.x.x.1` |

**학생이 집에서 실습하려면:**
1. `wifi_config.py`에 집 SSID와 비밀번호만 넣고 실행 (자동 IP로 바로 동작)
2. 정적 IP가 필요하면 Thonny 셸에서 `wlan.ifconfig()` 출력값을 확인 후 설정

#### 정적 IP 적용 방법 (main.py 수정)

정적 IP를 사용하려면 `wifi_config.py`에 IP 정보를 추가하고, `main.py`의 Wi-Fi 연결부에 한 줄을 추가합니다.

```python
# wifi_config.py에 추가
STATIC_IP = "172.30.6.10"    # 원하는 IP
SUBNET    = "255.255.0.0"    # 서브넷 마스크
GATEWAY   = "172.30.0.1"     # 게이트웨이
DNS       = "8.8.8.8"        # DNS 서버
```

```python
# main.py — Wi-Fi 연결 직후에 추가
from wifi_config import *

wlan.connect(WIFI_SSID, WIFI_PASSWORD)
while not wlan.isconnected():
    time.sleep(1)

# 정적 IP 설정 (이 줄 추가)
wlan.ifconfig((STATIC_IP, SUBNET, GATEWAY, DNS))

my_ip = wlan.ifconfig()[0]
print(f"IP 고정 완료: {my_ip}")
```

<Aside type="tip" title="정적 IP는 선택사항">
DHCP 자동 할당으로도 충분합니다. 정적 IP는 **매번 같은 주소로 접속하고 싶을 때**만 설정하세요. 처음 수업에서는 자동 할당을 권장합니다.
</Aside>
