<div class="mission-card">
  <div class="mission-header">
    <span class="mission-title">활동: 프롬프트 Wall</span>

  </div>
  <div class="mission-body">
    <div class="mission-desc">

      <ul>
        <li><strong>앞면 (초록):</strong> 가장 잘 먹힌 프롬프트 1개</li>
        <li><strong>앞면 (분홍):</strong> 가장 안 먹힌 프롬프트 1개</li>

      </ul>
      칠판에 "잘 먹힌 프롬프트" / "안 먹힌 프롬프트" 영역을 나누고 붙입니다.
    </div>
  </div>
</div>

<div class="teacher-script">
  <p>"패턴이 보이죠? 잘 먹힌 프롬프트들의 공통점이 뭘까요?"</p>
</div>

<div class="key-terms">
  <dl>
    <dt>잘 먹힌 프롬프트 패턴</dt>
    <dd>코드를 함께 붙여넣었다 / 원하는 결과를 구체적으로 말했다 / 제약 조건을 명시했다 ("MicroPython", "외부 라이브러리 없이") / 짧아도 핵심이 담겨 있었다</dd>
    <dt>안 먹힌 프롬프트 패턴</dt>
    <dd>코드 없이 요청만 보냈다 / "대충 바꿔줘" 식으로 모호했다 / MicroPython이라고 안 말해서 일반 Python 코드가 왔다 / 한 번에 너무 많은 것을 요청했다</dd>
  </dl>
</div>

<div class="class-scene">
  <div class="scene-line"><span class="speaker speaker-student">수아:</span> "저는 '예쁘게 해줘'라고 했을 때 안 먹혔고, '카드 배경을 그라디언트로, 둥근 모서리 16px, 그림자 효과'라고 했을 때 바로 됐어요."</div>
  <div class="scene-line"><span class="speaker speaker-student">지호:</span> "저는 에러 메시지를 통째로 복사해서 보내니까 AI가 바로 고쳐줬어요."</div>
  <div class="scene-line"><span class="speaker speaker-teacher">선생님:</span> "좋은 포인트예요. 구체적일수록, 맥락을 많이 줄수록 AI가 정확하게 응답해요."</div>
</div>

<div class="step-header">
  <span class="step-num">4</span>
  <span class="step-title">전체 토론: "진짜 설치한다면?"</span>
  <span class="step-time">44-48분</span>
</div>

<div class="teacher-script">
  <p>"만약 이 전광판을 <strong>진짜 학교 복도에 설치</strong>한다면 뭘 더 고려해야 할까요? 자유롭게 이야기해보세요."</p>
</div>

<div class="key-terms">
  <dl>
    <dt>전원</dt>
    <dd>USB로 24시간 연결? 배터리? → 안정적 전원 관리 필요</dd>
    <dt>화면</dt>
    <dd>스마트폰 대신 모니터 연결? → HDMI 출력 가능한 Raspberry Pi 4로 업그레이드</dd>
    <dt>안정성</dt>
    <dd>에러 나면 자동 재시작? → 워치독 타이머, 예외 처리 강화</dd>
    <dt>보안</dt>
    <dd>아무나 접속 가능한데? → 비밀번호 인증, 관리자 페이지 분리</dd>
    <dt>다중 접속</dt>
    <dd>30명이 동시에 보면? → Pico 한계, 서버 분리 필요</dd>
    <dt>API 키</dt>
    <dd>코드에 키가 노출되는 문제 → 환경변수 관리, 서버 프록시</dd>
    <dt>유지보수</dt>
    <dd>방학 동안 관리는? 졸업 후에는? → 자동화, 문서화, 인수인계</dd>
  </dl>
</div>

## 교사용 참고사항

### 평가 방법 제안

| 평가 항목 | 비중 | 기준 |
|----------|------|------|
| 전광판 완성도 | 30% | 4종 API 작동 여부, 에러 처리, 안정성 |
| 디자인 | 15% | 가독성, 정보 계층, 색상 조화, 모바일 최적화 |
| 창의적 확장 | 20% | 자기만의 API 추가, 독창적 기능, 문제 해결 |
| 바이브 코딩 역량 | 20% | 프롬프트 품질, AI 활용 과정, 디버깅 능력 |
| 발표 & 소통 | 15% | 핵심 전달, 경험 공유, 질의응답 |

<Aside type="tip" title="평가 팁">
바이브 코딩 역량(20%)은 결과물만이 아니라 **과정**을 봅니다. Best 프롬프트 3개를 제출받으면 학생이 AI를 어떻게 활용했는지 파악할 수 있습니다. 실패 후 대안을 찾은 학생에게도 높은 점수를 줄 수 있습니다.
</Aside>
