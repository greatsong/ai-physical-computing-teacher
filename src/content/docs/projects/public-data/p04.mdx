## 교사용 참고사항

### 이번 차시의 핵심

<div class="patrol-point">
  <ol>
    <li><strong>새 코드 제공 없음</strong> — 순수 바이브 코딩 차시</li>
    <li>교사는 순회하며 <strong>프롬프트 품질</strong>, <strong>디자인 원칙 적용 여부</strong> 피드백</li>
    <li>"기능 추가"가 아닌 "디자인 개선"에 집중하도록 안내</li>
    <li>CSS만 수정해도 충분히 큰 변화를 만들 수 있음을 강조</li>
  </ol>
</div>

### 평가 포인트 (참고)

| 평가 항목 | 관찰 기준 |
|----------|----------|
| 바이브 코딩 사이클 | 요청→확인→수정을 능숙하게 수행하는가 |
| 디자인 3원칙 | 가독성, 정보 계층, 가시성을 적용했는가 |
| 프롬프트 개선 | 점점 구체적으로 개선해가는 과정이 보이는가 |
| 문제 해결 | 에러 발생 시 AI를 활용해 해결하는가 |
