## 1. 이미 알고 있는 것에 이름 붙이기

<div class="outcome-preview">
  <p><strong>네트워크를 처음 배우는 사람은 "모르는 것을 배우는 게" 아니에요.</strong><br/>
  여러분이 매일 하는 행동에 <strong>정확한 이름을 붙이는 것</strong>입니다.</p>
</div>

여러분은 이미 네트워크를 쓰고 있습니다:

- 📱 학교 Wi-Fi에 폰을 연결해서 유튜브 본 적 있죠?
- 🖨️ 무선 프린터로 과제를 출력한 적 있죠?
- 📺 선생님이 노트북 화면을 전자칠판에 미러링하는 거 봤죠?

이 모든 것이 **네트워크** 위에서 일어나는 일입니다. 이제 이름을 붙여봅시다.

### 온도 26.5°C의 여행

센서 연구소 프로젝트를 예로 들어볼게요. 교실 온도 26.5°C라는 숫자가 어떻게 여러분의 스마트폰 화면에 나타나는지 따라가봅시다.

### 요청과 응답 — 카페 주문처럼!

Pico는 **웹 서버**입니다. 웹 서버란 "누군가 요청하면 응답해주는 프로그램"이에요.
카페에서 주문하는 것과 똑같습니다!

<div class="http-chat">

  </div>

<div class="teaching-tip">
  <p><strong>"실시간"의 비밀:</strong> 처음에 HTML을 1번 받고, 그 다음부터는 <strong>JSON 데이터만</strong> 반복 요청합니다. 매번 전체 페이지를 새로 받지 않아요! 네이버 실시간 검색어가 새로고침 없이 바뀌는 것도 같은 원리예요.</p>
</div>

## 핵심 용어 정리

<div class="key-terms">
  <dl>
    <dt>LAN</dt>
    <dd>Local Area Network. <strong>같은 공유기에 연결된 기기들의 모임</strong>. 아파트 비유의 "같은 단지".</dd>
    <dt>AP (Access Point)</dt>
    <dd>Wi-Fi 신호를 쏘는 쪽. 공유기, 또는 AP 모드의 Pico. <strong>"카페 사장님"</strong> 역할.</dd>
    <dt>STA (Station)</dt>
    <dd>Wi-Fi 신호를 받는 쪽. 노트북, 스마트폰, STA 모드의 Pico. <strong>"카페 손님"</strong> 역할.</dd>
    <dt>IP 주소</dt>
    <dd>네트워크 안에서 각 기기의 고유 주소. 아파트의 호수. <strong>192.168.1.xxx</strong> 형태.</dd>
    <dt>SSID</dt>
    <dd>Wi-Fi 이름. "School_WiFi", "DragonRadar" 등. 스마트폰에서 보이는 Wi-Fi 목록.</dd>
    <dt>웹 서버</dt>
    <dd>누군가 IP로 접속하면 웹 페이지(HTML)를 보내주는 프로그램. Pico가 이 역할을 합니다!</dd>
    <dt>JSON</dt>
    <dd>기계가 읽기 쉬운 데이터 형식. <code>{`{"temp":26.5}`}</code> 같은 모양. 실시간 업데이트에 사용.</dd>
    <dt>fetch()</dt>
    <dd>브라우저가 서버에 "데이터 주세요!"하고 요청하는 JavaScript 함수. 실시간의 핵심 도구.</dd>
  </dl>
</div>
