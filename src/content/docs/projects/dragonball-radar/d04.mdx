## 교사용 참고사항

### 바이브코딩 수업 운영 핵심

<div class="patrol-point">
  <ol>
    <li>교사는 코드를 직접 도와주기보다, <strong>AI에게 어떻게 요청할지</strong> 안내하는 역할에 집중하세요.</li>
    <li>완성도보다 <strong>과정(시행착오, 프롬프트 개선)</strong>이 중요합니다.</li>
    <li>1~3차시 기초 지식이 AI 코드 검증의 핵심입니다 -- 이 연결을 학생들이 체감하게 해주세요.</li>
    <li>에러가 학습 기회입니다 -- 에러 메시지를 AI에게 보여주는 습관을 강조하세요.</li>
  </ol>
</div>

### 차별화된 학생 지원

| 학생 유형 | 지원 방법 |
|-----------|----------|
| 프롬프트를 모호하게 쓰는 모둠 | 좋은 프롬프트 4요소 다시 안내 (환경, 기능 목록, 기존 코드, 제약 조건) |
| AI 코드를 검증 없이 그대로 쓰는 모둠 | "AP 설정이 있어?", "이동평균 코드가 있어?" 질문으로 검증 유도 |
| 빠르게 끝난 모둠 | 보너스 미션 (confetti, 효과음, 통계) 추천 |
| 에러에 막힌 모둠 | "에러 메시지를 AI에게 보냈어?" 확인 후 프롬프트 개선 안내 |
| 미션 1도 못 끝낸 모둠 | 3차시 기본 코드 백업 제공 → 미션 1부터 다시 시도 |

### 자주 나오는 기술 문제
