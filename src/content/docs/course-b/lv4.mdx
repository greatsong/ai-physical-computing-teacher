## 준비물 체크리스트

<div class="prep-checklist">
  <h5>하드웨어 (팀별 1세트)</h5>
  <ul>
    <li>Raspberry Pi Pico 2 WH + Grove Shield</li>
    <li>DHT20 온습도 센서 (I2C, 0x38)</li>
    <li>빛 센서 (ADC, GP26 — A0)</li>
    <li>먼지 센서 GP2Y (ADC, GP28 — A2)</li>
    <li>사운드 센서 (ADC, GP27 — A1)</li>
    <li>OLED 디스플레이 SSD1306 (I2C, 0x3C)</li>
    <li>USB 데이터 케이블</li>
  </ul>
  <h5>소프트웨어</h5>
  <ul>
    <li>Thonny IDE — MicroPython 코드 작성 및 업로드</li>
    <li>크롬 브라우저 — Supabase 접속, 대시보드 확인</li>
  </ul>
  <h5>계정 / 네트워크</h5>
  <ul>
    <li>Supabase 무료 계정 (Free plan) — 사전에 교사가 1개 프로젝트를 만들어 흐름을 숙지할 것</li>
    <li>학교 WiFi (2.4GHz 대역) — Pico는 5GHz를 지원하지 않음, 사전에 반드시 확인</li>
    <li>GitHub 계정 (선택) — Supabase 가입 시 편리</li>
  </ul>
  <h5>수업 전 확인</h5>
  <ul>
    <li>supabase.com이 학교 네트워크에서 접속 가능한지 확인</li>
    <li>학교 WiFi SSID와 비밀번호를 학생에게 공유할 준비</li>
    <li>Pico가 학교 WiFi에 연결되는지 교사 기기로 사전 테스트</li>
    <li>각 팀의 센서가 정상 작동하는지 간단 테스트 (I2C scan 등)</li>
  </ul>
</div>

<div class="step-header">
  <span class="step-num">2</span>
  <span class="step-title">테이블 생성</span>
  <span class="step-time">15분</span>
</div>

<div class="teacher-script">
  <p>"프로젝트가 만들어졌으면, 이제 데이터를 저장할 <strong>테이블</strong>을 만들어야 해요."</p>
  <p>"테이블은 엑셀 시트라고 생각하세요. 열(column)에 이름을 정하고, 데이터가 한 행씩 쌓이는 거예요."</p>
  <p>"왼쪽 메뉴에서 <strong>SQL Editor</strong> 아이콘을 클릭하세요. 'New query'를 누르고, 교재의 SQL 코드를 <strong>전체 복사</strong>해서 붙여넣으세요."</p>
  <p>"Run 버튼을 누르면 끝! 'Success. No rows returned'가 나오면 정상이에요. 테이블을 '만드는' 명령이라 돌려줄 데이터가 없는 거예요."</p>
</div>

<div class="teacher-script">
  <p>"테이블의 각 컬럼을 설명해 줄게요."</p>
  <p>"<strong>id</strong>는 BIGSERIAL — 자동으로 1, 2, 3... 번호가 매겨집니다. 엑셀의 행 번호 같은 거예요."</p>
  <p>"<strong>created_at</strong>은 TIMESTAMPTZ — 데이터가 들어온 시각이 자동으로 기록돼요."</p>
  <p>"<strong>temp, humi</strong>는 REAL — 소수점이 있는 숫자, 즉 실수(float)예요. 온도 24.5도, 습도 65.3% 같은 값을 저장합니다."</p>
  <p>"<strong>co2</strong>는 INTEGER — 정수예요. CO2는 800ppm처럼 정수로 충분합니다."</p>
  <p>"<strong>device_id</strong>는 TEXT — 문자열이에요. Pico가 여러 대일 때 구분하는 이름표입니다."</p>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"컬럼 타입이 뭐예요?"</div>
    <span class="label">대응</span>
    <span class="value">REAL = 소수점 숫자(온도, 습도), INTEGER = 정수(CO2), TEXT = 글자(기기 이름), TIMESTAMPTZ = 날짜+시간. 엑셀에서 셀 형식을 '숫자'나 '텍스트'로 지정하는 것과 같습니다</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"ERROR: relation already exists"</div>
    <span class="label">원인</span>
    <span class="value">이미 같은 이름의 테이블이 존재함 (아마 두 번 실행)</span>
    <span class="label">대응</span>
    <span class="value">정상입니다! 이미 테이블이 만들어져 있다는 뜻이니 그대로 진행하세요</span>
  </div>
</div>

<div class="teaching-tip">
  <p>SQL이 어렵게 느껴질 수 있지만, 이 수업에서 학생이 직접 SQL을 작성할 필요는 없습니다. 교재의 코드를 복사+붙여넣기하면 됩니다. "SQL은 데이터베이스에 명령을 내리는 언어"라고 한 줄 설명하고, CREATE TABLE이 "엑셀에서 시트 만들기"에 대응된다고 비유하면 충분합니다.</p>
</div>

<div class="teacher-script">
  <p>"테이블을 만들었으면, 이제 RLS 보안 설정을 합니다. <strong>새 쿼리</strong>를 열고, RLS SQL 코드를 붙여넣고 Run 하세요."</p>
  <p>"RLS는 '누가 데이터를 볼 수 있고, 추가할 수 있는지' 정하는 규칙이에요. 우리는 누구나 읽고 추가할 수 있지만, 삭제는 못하게 설정합니다."</p>
</div>

<div class="step-header">
  <span class="step-num">3</span>
  <span class="step-title">REST API 연동 코드</span>
  <span class="step-time">10분</span>
</div>

<div class="teacher-script">
  <p>"이제 가장 중요한 단계입니다. Pico가 Supabase에 데이터를 보내려면 <strong>두 가지 열쇠</strong>가 필요해요."</p>
  <p>"왼쪽 메뉴 맨 아래 <strong>Project Settings</strong>(톱니바퀴)를 클릭하고, <strong>API</strong>를 선택하세요."</p>
  <p>"첫 번째: <strong>Project URL</strong> — 'https://xxxxx.supabase.co' 형태의 주소를 복사하세요."</p>
  <p>"두 번째: <strong>anon public key</strong> — 'eyJhbGci...'로 시작하는 매우 긴 문자열을 복사하세요. 전체를 복사해야 합니다!"</p>
  <p>"<strong>주의!</strong> 'service_role secret' 키는 절대 복사하지 마세요. 그건 관리자 전용이에요."</p>
</div>

<div class="teacher-script">
  <p>"이 두 가지를 Pico 코드에 넣어야 합니다. 교재의 코드를 보세요. 헤더(headers)에 세 가지가 들어갑니다."</p>
  <p>"<strong>apikey</strong>: 복사한 anon 키"</p>
  <p>"<strong>Authorization</strong>: 'Bearer ' + 같은 anon 키 — 앞에 'Bearer '를 붙이는 게 포인트예요"</p>
  <p>"<strong>Content-Type</strong>: 'application/json' — 데이터 형식이 JSON이라고 알려주는 거예요"</p>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"URL이랑 KEY를 어디서 찾아요?"</div>
    <span class="label">대응</span>
    <span class="value">Settings(톱니바퀴) → API → Project URL과 Project API keys 섹션. 화면을 보여주며 함께 찾으세요</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"키가 너무 길어서 다 복사가 안 돼요"</div>
    <span class="label">대응</span>
    <span class="value">키 옆의 복사 버튼(아이콘)을 클릭하면 전체가 자동 복사됩니다. 드래그로 선택하면 놓칠 수 있어요</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>이 시점에 순회하며 확인:</strong></p>
  <ol>
    <li>테이블 생성 SQL이 성공했는가 (Table Editor에서 sensor_data 테이블이 보이는가)</li>
    <li>RLS 정책이 설정되었는가</li>
    <li>Project URL과 anon key를 정확히 복사했는가</li>
    <li>service_role 키를 실수로 복사하지 않았는가</li>
  </ol>
</div>

<div class="teacher-script">
  <p>"1차시 정리! 오늘 여러분은 클라우드 데이터베이스를 만들었어요. 테이블, 보안, API 키 — 이 세 가지가 클라우드의 기본 구성요소입니다."</p>
  <p>"다음 시간에는 실제로 센서 데이터를 수집하고, Supabase에 전송한 뒤, 규칙 기반 분류까지 해볼 거예요."</p>
</div>

    <span class="label">대응</span>
    <span class="value">Pico는 <strong>2.4GHz WiFi만</strong> 지원합니다. 학교 WiFi가 5GHz 전용이면 핫스팟을 사용하세요. SSID 철자를 정확히 확인하세요</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"전송 실패 반복"</div>
    <span class="label">원인</span>
    <span class="value">Supabase URL 또는 KEY 오류</span>
    <span class="label">대응</span>
    <span class="value">Settings → API에서 URL과 anon key를 다시 복사하세요. URL 끝에 슬래시(/)가 있으면 제거하세요</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"WiFi 연결에 시간이 너무 오래 걸려요"</div>
    <span class="label">원인</span>

    <span class="label">대응</span>

<div class="patrol-point">

<div class="teaching-tip">

<div class="step-header">
  <span class="step-num">2</span>
  <span class="step-title">규칙 기반 분류 로직</span>
  <span class="step-time">15분</span>
</div>

<div class="teacher-script">
  <p>"데이터가 잘 들어가고 있죠? 이제 이 데이터를 보고 <strong>컴퓨터가 판단</strong>하게 만들어 봅시다."</p>
  <p>"교재에 있는 air_quality_index 함수를 보세요. 100점에서 시작해서, 조건에 따라 점수를 깎는 방식이에요."</p>
  <p>"예를 들어: 온도가 28도 이상이고 습도가 70% 이상이면? '불쾌' 등급이에요. 이것을 코드로 쓰면:"</p>
  <p>"<code>if temp > 28 and humi > 70: label = '불쾌'</code>"</p>
  <p>"이게 바로 <strong>규칙 기반 분류</strong>예요. 그리고 이것이 머신러닝의 가장 기본 형태입니다."</p>
</div>

<div class="student-voice">
  <span class="bubble">"if문이 머신러닝이에요?"</span>
  <span class="bubble">"그럼 지금까지 한 거 다 ML이었어요?"</span>
</div>

<div class="teacher-script">
  <p>"좋은 질문이에요! 차이가 있어요. 지금 우리가 하는 건 <strong>사람이 규칙을 정하는</strong> 거예요. '28도 이상이면 덥다'는 규칙을 우리가 만들었잖아요."</p>
  <p>"진짜 머신러닝은 이 규칙을 <strong>데이터에서 자동으로 찾아내는</strong> 거예요. 하지만 원리는 같습니다 — 데이터를 보고 분류하는 것."</p>
  <p>"규칙 기반은 ML의 출발점이에요. 먼저 규칙으로 해보고, 나중에 데이터가 충분히 쌓이면 자동 학습으로 발전시키는 거죠."</p>
</div>

<div class="teaching-tip">
  <p>"if문 = ML인가요?"라는 질문이 나오면, "규칙 기반 분류는 ML의 가장 단순한 형태"라고 설명하세요. ML의 핵심은 "데이터를 입력받아 분류/예측하는 것"이고, 규칙을 사람이 만들면 rule-based, 데이터에서 자동으로 학습하면 supervised learning입니다. 이 수업에서는 rule-based부터 시작하여 ML의 개념을 체험하는 것이 목표입니다. AQI 점수 체계에서 CO2 임계값(>2000: -50, >1500: -40, >1000: -30, >800: -15)이 계단식으로 설정된 이유를 함께 토론하면 "왜 이 값인지"를 생각하게 됩니다.</p>
</div>

<div class="teacher-script">
  <p>"이제 분류 함수를 코드에 추가하세요. AQI 점수를 계산하고, 등급을 매기는 함수입니다."</p>
  <p>"Good(80점 이상), Fair(60점 이상), Poor(40점 이상), Bad(40점 미만) — 이 네 가지 등급으로 분류합니다."</p>
  <p>"분류된 등급을 Supabase에 함께 전송하면, 대시보드에서 등급별 필터링도 할 수 있어요."</p>
</div>

<div class="teaching-tip">
  <p>먼지 센서(GP2Y)의 정확도 한계를 미리 안내하세요. GP2Y는 전압 기반 근사식(<code>(voltage - 0.6) * 170</code>)을 사용하며, 전문 장비와 오차가 큽니다. ADC 핀은 GP26=A0(빛), GP27=A1(소리), GP28=A2(먼지)로 배정되어 있음을 확인하세요. "전문 미세먼지 측정기와 비교하면 차이가 있지만, 상대적 변화(먼지가 많아지는 추세)는 잘 잡아냅니다"라고 설명하세요.</p>
</div>
