    <li>PC에 Python 설치 (python.org) + pip install flask flask-cors matplotlib</li>
    <li>AI 챗봇 접속 확인 (ChatGPT 또는 Claude)</li>
  </ul>
  <h5>수업 전 확인 (중요!)</h5>
  <ul>
    <li>교사 PC에 Python + Flask가 정상 설치되어 있는지 사전 테스트</li>
    <li>교실 WiFi가 기기 간 통신을 허용하는지 확인 — 차단 시 핸드폰 핫스팟 대안 준비</li>
    <li>교사 PC의 IP 주소 확인: Windows <code>ipconfig</code> / Mac 시스템 설정 → WiFi → 세부사항</li>

    <li>학생 수만큼 WiFi SSID와 비밀번호를 칠판에 적어 둘 것</li>

## 1차시: Flask 서버 설치 + 기본 라우트 (50분)

### 도입 (5분)

<div class="teacher-script">
  <p>"Lv.2까지 우리는 Pico 안에서만 데이터를 저장했어요. CSV 파일에 기록하고, 나중에 Thonny로 다운로드해서 PC에서 분석했죠."</p>
  <p>"그런데 이 방식에는 불편한 점이 있어요. USB를 꽂아야 하고, 측정하는 동안 PC에서 실시간으로 확인할 수가 없죠."</p>
  <p>"오늘부터 3차시에 걸쳐 <strong>WiFi 실시간 통신 시스템</strong>을 만듭니다. Pico가 센서 데이터를 무선으로 PC 서버에 전송하고, 웹 브라우저에서 실시간 대시보드를 확인할 수 있어요."</p>
  <p>"이건 실제로 IoT 회사에서 쓰는 방식과 동일합니다. <strong>클라이언트-서버 구조</strong>라고 합니다."</p>
</div>

<div class="student-voice">
  <span class="bubble">"WiFi로 데이터를 보낸다고요?"</span>
  <span class="bubble">"웹 브라우저에서 볼 수 있어요?"</span>
  <span class="bubble">"Flask가 뭐예요?"</span>

### 전개 (40분)

<div class="step-header">
  <span class="step-num">1</span>
  <span class="step-title">Python + Flask 설치</span>
  <span class="step-time">10분</span>
</div>

<div class="teacher-script">
  <p>"먼저 PC에 Flask를 설치합니다. 터미널(명령 프롬프트)을 열고 아래 명령어를 입력하세요."</p>
  <p>"Windows는 시작 메뉴에서 'cmd'를 검색, Mac은 Spotlight에서 '터미널'을 검색하면 됩니다."</p>
  <p><code>pip install flask flask-cors</code></p>
  <p>"'Successfully installed'가 나오면 성공입니다. 만약 'pip가 안 된다'고 나오면, Python이 PATH에 등록되지 않은 거예요."</p>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"pip가 안 돼요" / "'pip' is not recognized"</div>
    <span class="label">원인</span>
    <span class="value">Python 설치 시 "Add Python to PATH" 체크를 안 한 경우</span>
    <span class="label">대응</span>
    <span class="value">Python 재설치 시 PATH 체크박스 반드시 체크. 또는 <code>py -m pip install flask flask-cors</code> 시도. Mac은 <code>pip3</code> 사용</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"Permission denied" / 권한 오류</div>
    <span class="label">원인</span>
    <span class="value">시스템 Python에 직접 설치하려는 경우 (Mac에서 자주 발생)</span>
    <span class="label">대응</span>
    <span class="value"><code>pip3 install --user flask flask-cors</code> 또는 관리자 권한으로 터미널 실행</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>이 시점에 순회하며 확인:</strong></p>
  <ol>
    <li>모든 팀의 PC에 Flask가 정상 설치되었는가 ("Successfully installed..." 메시지 확인)</li>
    <li>터미널에서 <code>python -c "import flask; print(flask.__version__)"</code>이 실행되는가</li>
    <li>설치가 안 되는 팀은 교사 PC에서 서버를 공유하는 것을 대안으로 안내</li>
  </ol>
</div>

<div class="step-header">
  <span class="step-num">2</span>
  <span class="step-title">Flask 서버 코드 작성 + 실행</span>
  <span class="step-time">15분</span>
</div>

<div class="teacher-script">
  <p>"학생 교재에 있는 server.py 코드를 그대로 복사해서 파일로 저장합니다."</p>
  <p>"중요한 포인트가 하나 있어요. 서버 코드 맨 아래에 <code>app.run(host='0.0.0.0', port=5000)</code>이라고 써 있죠? 여기서 <strong>'0.0.0.0'이 핵심</strong>입니다."</p>
  <p>"만약 'localhost'나 '127.0.0.1'로 쓰면 자기 PC에서만 접속 가능하고, 다른 기기(Pico)에서 보낼 수 없어요. 반드시 '0.0.0.0'이어야 합니다."</p>
  <p>"저장 후 터미널에서 <code>python server.py</code>를 실행하면, 서버 IP 주소가 화면에 나타납니다. 이 IP를 메모하세요."</p>
</div>

<div class="student-voice">
  <span class="bubble">"server.py를 어디에 저장해요?"</span>
  <span class="bubble">"실행했더니 IP 주소가 나왔어요!"</span>
  <span class="bubble">"Mac인데 포트 5000이 이미 사용 중이래요"</span>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"Address already in use" (Mac에서 포트 5000 충돌)</div>
    <span class="label">원인</span>
    <span class="value">macOS Monterey 이상에서 AirPlay Receiver가 포트 5000을 사용</span>
    <span class="label">대응</span>
    <span class="value">server.py에서 <code>port=5000</code>을 <code>port=5001</code>로 변경. 이후 Pico 코드의 SERVER 주소에서도 포트를 5001로 맞출 것</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"server.py를 어디에 저장하는지 모르겠어요"</div>
    <span class="label">원인</span>
    <span class="value">파일 저장 경로와 터미널 작업 디렉토리가 다른 경우</span>
    <span class="label">대응</span>
    <span class="value">바탕화면에 'pico-server' 폴더를 만들고 그 안에 저장. 터미널에서 <code>cd Desktop/pico-server</code> 후 실행</span>
  </div>
</div>

<div class="teaching-tip">
  <p><code>host='0.0.0.0'</code>은 이 수업에서 가장 자주 빠뜨리는 부분입니다. 학생 교재 코드에는 이미 포함되어 있지만, AI에게 Flask 코드를 요청하면 <code>host='127.0.0.1'</code>로 생성하는 경우가 많습니다. 칠판에 크게 써두세요: <strong>"0.0.0.0 = 모든 기기 접속 허용"</strong></p>
</div>

<div class="step-header">
  <span class="step-num">3</span>
  <span class="step-title">IP 주소 확인 + 브라우저 테스트</span>
  <span class="step-time">10분</span>
</div>

<div class="teacher-script">
  <p>"서버가 실행되고 있죠? 이제 IP 주소를 확인합니다."</p>
  <p>"Windows 팀은 <strong>새 터미널 창</strong>을 열고(서버 터미널은 닫지 마세요!) <code>ipconfig</code>를 입력합니다. 'IPv4 주소'를 찾으세요. 보통 192.168로 시작합니다."</p>
  <p>"Mac 팀은 시스템 설정 → WiFi → 연결된 네트워크 옆 ⓘ 클릭 → IP 주소를 확인합니다."</p>
  <p>"확인한 IP로 브라우저에 <code>http://IP주소:5000</code>을 입력해 보세요. 대시보드 화면이 보이면 성공입니다!"</p>
</div>

<div class="student-voice">
  <span class="bubble">"192.168.0.15가 제 IP예요!"</span>
  <span class="bubble">"브라우저에 대시보드가 떴어요!"</span>
  <span class="bubble">"아직 데이터가 0개라고 나와요"</span>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"브라우저에서 접속이 안 돼요"</div>
    <span class="label">원인</span>
    <span class="value">방화벽이 포트를 차단하거나, 서버가 실행 중이 아닌 경우</span>
    <span class="label">대응</span>
    <span class="value">서버 터미널이 열려 있는지 확인. Windows 방화벽 알림에서 "허용" 클릭. IP 주소와 포트 번호 다시 확인</span>
  </div>
</div>

<div class="patrol-point">
  <p><strong>순회 시 확인:</strong></p>
  <ol>
    <li>Flask 서버가 실행 중인 터미널이 열려 있는가 (닫으면 서버도 종료됨)</li>
    <li>브라우저에서 대시보드 페이지가 정상 표시되는가</li>
    <li>각 팀의 PC IP 주소를 메모해 두었는가 (2차시에 필요)</li>
  </ol>
</div>

<div class="step-header">

<div class="teacher-script">

  <p>"첫째, <code>WIFI_SSID</code>에 WiFi 이름을 넣으세요. 둘째, <code>WIFI_PASS</code>에 비밀번호를 넣으세요. 셋째, <code>SERVER</code>에 지난 시간 메모한 PC의 IP 주소를 넣으세요."</p>
  <p>"WiFi 연결 타임아웃은 20초입니다. 연결되면 Pico의 IP 주소가 출력되고, 10초마다 서버에 데이터를 전송합니다."</p>
</div>

<div class="student-voice">
  <span class="bubble">"WiFi 연결 성공이라고 나왔어요!"</span>
  <span class="bubble">"서버에서 '수신' 메시지가 뜨고 있어요!"</span>
  <span class="bubble">"연결 실패래요..."</span>
  <span class="bubble">"서버에서 안 받아요"</span>
</div>

<div class="trouble-cards">
  <div class="trouble-card">
    <div class="symptom">"WiFi 연결 실패!" — 20초 타임아웃</div>
    <span class="label">원인</span>
    <span class="value">SSID/비밀번호 오타, 5GHz WiFi 사용, 또는 WiFi 신호 약함</span>
    <span class="label">대응</span>
    <span class="value">SSID 철자 확인 (대소문자 구분). Pico는 <strong>2.4GHz만 지원</strong> — 5GHz WiFi는 안 됨. 핸드폰 핫스팟(2.4GHz 설정)을 대안으로 사용</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"서버에서 안 받아요" / "Connection refused"</div>
    <span class="label">원인</span>
    <span class="value">Pico와 PC가 같은 네트워크에 있지 않거나, 서버가 실행 중이 아닌 경우</span>
    <span class="label">대응</span>
    <span class="value">① Pico와 PC가 <strong>같은 WiFi</strong>에 연결되어 있는지 확인 ② Flask 서버 터미널이 열려 있는지 확인 ③ IP 주소가 정확한지 확인 ④ 학교 WiFi가 기기 간 통신을 차단하면 핫스팟 사용</span>
  </div>
  <div class="trouble-card">
    <div class="symptom">"urequests 모듈이 없어요"</div>

## 기술 참조 (교사용)

### Flask 서버 핵심 설정

| 항목 | 값 | 주의사항 |
|------|-----|---------|
| Flask 설치 | `pip install flask flask-cors` | Mac: `pip3` 사용 |
| 서버 실행 | `app.run(host='0.0.0.0', port=5000)` | **반드시** `0.0.0.0` — localhost 사용 금지 |
| PC IP 확인 | Windows: `ipconfig` / Mac: 시스템 설정 → WiFi | IPv4 주소 (192.168.x.x) |
| Mac 포트 충돌 | 포트 5000 → 5001로 변경 | AirPlay Receiver가 5000 사용 |

### Pico WiFi 핵심 설정

| 항목 | 값 | 주의사항 |
|------|-----|---------|
| WiFi 주파수 | 2.4GHz만 지원 | 5GHz WiFi는 연결 불가 |
| WiFi 타임아웃 | 20초 (0.5초 x 40회) | 핫스팟 사용 시 연결 시간 더 걸릴 수 있음 |
| 데이터 전송 | `urequests.post(url, json=data)` | 반드시 `res.close()` 호출 (메모리 해제) |
| I2C 설정 | `I2C(1, sda=Pin(6), scl=Pin(7))` | 항상 I2C1 사용 |
| MPU6050 | 주소 0x68, 슬립 해제: `0x6B` 레지스터에 `0x00` | 초기화 안 하면 데이터 0 |
| CSV 헤더 | `os.stat()` 또는 `os.listdir()`로 존재 확인 | 새 파일일 때만 헤더 작성 |

### 네트워크 요구사항

- Pico와 PC가 **같은 WiFi 네트워크**에 연결되어 있어야 함
- 학교 WiFi가 기기 간 통신(AP isolation)을 차단하면 핸드폰 핫스팟 사용
- 서버 PC의 방화벽에서 포트 5000(또는 5001)을 허용해야 함
