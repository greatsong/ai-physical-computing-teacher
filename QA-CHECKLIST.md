# 바이브 코딩 프로젝트 교재 — 품질 검수 기준

> **검수 비중**: 하드웨어/센서/회로 정확성 **80%** | 교수 설계·콘텐츠·시각 **20%**

---

## 1. 하드웨어·센서·회로 정확성 (80%)

### 1-1. 핀 번호 및 배선 정확성 (최우선)
- [ ] Pico 2 WH는 **Micro USB** (USB-C 아님)로 전체 문서에서 통일되었는가
- [ ] 센서별 VCC, GND, 신호선(SIG/SDA/SCL) 연결 핀 번호가 정확한가
- [ ] I2C 핀: SDA=GP4(6번), SCL=GP5(7번) 으로 일관되는가
- [ ] ADC 핀: GP26(31번)=ADC0, GP27(32번)=ADC1, GP28(34번)=ADC2 정확한가
- [ ] 3V3(36번), GND(38번) 등 전원 핀 번호가 정확한가
- [ ] 센서별 동작 전압이 3.3V/5V 구분되어 기재되었는가

### 1-2. 센서 스펙 및 프로토콜 정확성
- [ ] DHT20: I2C 주소 0x38, 4선 연결(VCC/GND/SDA/SCL) 정확한가
- [ ] 조도 센서: 아날로그(ADC), 3선 연결(VCC/GND/SIG), GP26 정확한가
- [ ] 소리 센서: 아날로그(ADC), 3선 연결, 피크 샘플링 방식 정확한가
- [ ] PIR 센서: 디지털 출력, 3선 연결, 감지 시 HIGH 출력 정확한가
- [ ] SCD30: I2C 주소 0x61, 4선, CO₂ 범위 400~10000ppm 정확한가
- [ ] LPS22HB: I2C 주소 0x5C(또는 0x5D), 정확도 ±0.025 hPa 정확한가
- [ ] 각 센서의 측정 범위, 정확도, 응답 시간이 사실과 일치하는가

### 1-3. MicroPython 코드 정확성
- [ ] `from machine import Pin, ADC, I2C` 등 import문이 MicroPython에서 유효한가
- [ ] `ADC(Pin(26))`, `I2C(0, sda=Pin(4), scl=Pin(5))` 등 하드웨어 초기화가 정확한가
- [ ] I2C 주파수 설정이 센서별로 적절한가 (SCD30: 50kHz, LPS22HB: 100kHz)
- [ ] 센서 드라이버 코드의 레지스터 주소, CRC 계산, 바이트 변환이 정확한가
- [ ] `read_u16()` 반환값 범위(0~65535)가 정확히 기재되었는가
- [ ] `gc.collect()` 호출이 메모리 관리를 위해 적절히 배치되었는가
- [ ] 웹서버 코드(`socket.socket`, `bind`, `listen`)가 MicroPython에서 동작하는가

### 1-4. 배선도 시각 자료
- [ ] 배선도가 ASCII art 대신 **styled HTML**로 표현되었는가
- [ ] 센서 핀 → Pico 핀 매핑이 텍스트와 배선도에서 일치하는가
- [ ] 선 색상 구분 안내가 있는가 (빨강=VCC, 검정=GND, 노랑=SIG 등)
- [ ] 브레드보드 배치가 현실적인가 (핀 간섭, 접촉불량 가능성 고려)

### 1-5. API 및 네트워크 정확성
- [ ] 나이스 API 엔드포인트 URL, 파라미터명이 정확한가
- [ ] 기상청 API: 격자좌표(nx,ny), base_time, 카테고리 코드(T1H,REH,PTY,POP,SKY) 정확한가
- [ ] 에어코리아 API: HTTP(HTTPS 아님!) 사용, 측정소명, 필드명 정확한가
- [ ] 서울시 버스 API: arsId, XML 태그명(rtNm,arrmsg1,traTime1) 정확한가
- [ ] `fetch_https()` vs `fetch_http()` 사용이 API별로 올바른가
- [ ] Wi-Fi는 2.4GHz만 지원된다고 명시되었는가

### 1-6. 하드웨어 안전 및 주의사항
- [ ] 전원 연결 순서(USB 먼저 → 센서 연결) 안내가 있는가
- [ ] SCD30 등 고가 센서의 취급 주의 안내가 있는가
- [ ] 보조배터리 사용 시 주의사항(기압 센서 탐구)이 있는가
- [ ] 센서 오작동 시 "배선 확인 → I2C 스캔 → 리셋" 순서가 안내되었는가
- [ ] ENOMEM 에러 대응법이 안내되었는가 (gc.collect, HTML 간소화)

## 2. 교수 설계 (10%)

### 2-1. 수업 흐름
- [ ] 50분 타임라인이 현실적인가 (코드 배포+실행 최소 5분 확보)
- [ ] 바이브 코딩 미션에 충분한 시간이 배정되었는가
- [ ] FULL/LIGHT 프롬프트가 모든 미션에 제공되는가
- [ ] 미션 난이도가 ★→★★★★★ 순으로 올라가는가

### 2-2. 교사 스크립트
- [ ] 교사 발화가 자연스럽고 구체적인가
- [ ] 예상 학생 반응/오류 상황 대응이 포함되었는가
- [ ] 트러블슈팅 표(증상→원인→해결)가 포함되었는가
